## 解耦保障

**核心原则：贡献者的工作不会因为 V1/V2 的变化而失效。**

怎么做到的：

1. **SDK 是契约层**：贡献者面向 7 个 Protocol 编程。协议层密封，实现层可变。只要 Protocol 接口不变，贡献者的代码就不会坏。
2. **研究任务（A/H 系列）输出是文档和实验**：不依赖具体代码路径，架构原则不变则研究有效。
3. **内容任务（C/D 系列）输出是文字**：基于设计思想，不基于代码。
4. **应用任务（S 系列）基于 SDK 公开 API**：`pip install` → `from towow import ...` → 构建。核心代码如何变，不影响他们。
5. **商业任务（P 系列）输出是运营内容**：不依赖技术实现细节。
