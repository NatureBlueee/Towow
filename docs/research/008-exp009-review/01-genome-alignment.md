# 基因组对齐审查

**审查者视角**: 产品/世界观守护者
**审查对象**: EXP-009 结晶实验 (SIM-001 + SIM-002)
**基准文档**: 通爻协议基因组 v0.3
**日期**: 2026-02-18

---

## 结论摘要

模块二的结晶协议在概念定位上与基因组存在一个真实的张力，但这个张力不是偏离，而是基因组自身尚未解决的开放问题（第8条："可见性和行动之间是否需要一个最小的通道？"）的一种回答。实验证明了催化机制在跨语义翻译上的价值（v1 从0条到24条，GT发现率从30%到80%），但同时暴露了一个产品层面的风险：**方案生成步骤把通爻从"发现层协议"拉向了"咨询服务"**，而plan.md的详尽程度说明这个滑坡已经在发生。最终输出的70万联合服务包方案，更像三个顾问的项目提案，而不是"让参与者看见彼此价值后自主决定下一步"。如果不在协议层面明确结晶的终止线，模块二会逐步吞噬基因组为WOWOK和应用层预留的空间。

---

## 详细审查

### 1. 模块二在基因组中的位置

**基因组原文**: "通爻只做一件事：让意图以趋近于零的成本找到相关意图，建立可见性。可见性建立之后——参与者怎么交流、怎么协作、怎么交付、怎么付款——不是通爻的事。"（SS3）

**设计文档的定位**: "模块二解决：房间里的人如何发现彼此的价值。"

这两句话之间有一个微妙但关键的缝隙。

模块一做的是"谁应该在一个房间"——这无争议地属于可见性。模块二做的是"房间里的人如何发现彼此的价值"——这到底是可见性的深化，还是已经跨入了可见性之后的领域？

**我的判断：前半段是，后半段不是。**

催化Agent做的"跨语义翻译"（"A说的X和B说的Y可能在说同一件事"）确实是在扩展可见性。模块一通过向量距离告诉你"你们应该在一起"，但没告诉你为什么。催化通过翻译把"为什么"变得可见。这是从"粗粒度可见性"到"细粒度可见性"的深化，完全符合基因组。

但方案生成步骤——让系统产出"谁做什么、收多少钱、什么时间表"——这已经不是可见性了。这是行动设计。基因组明确说这不是通爻的事。

设计文档自己也意识到了这个问题，它在V1映射表中写"Plan不再被生成，自然涌现"。但实验的实际做法是：结晶循环之后，有一个独立的"方案生成器"步骤，它读取全部对话记录，产出结构化的协作方案。这不是"自然涌现"，这是一个系统功能。

**核心问题：结晶的循环部分（催化+翻译）属于通爻协议层。方案生成步骤属于应用层。两者目前被打包在"模块二"里，边界模糊。**

### 2. 方案是否服务于原始张力

触发事件：东南亚制造业CEO要品牌转型，缺品牌叙事、组织转型、数字化三种能力。

plan.md 产出的方案：一个3个月、70万人民币的联合咨询项目，含阶段划分、角色定义、定价分配、依赖关系、退出机制。

**方案确实直接回应了原始张力。** CEO说"我缺一个能看到全局的人"，方案给了他三个人的组合以及他们如何协同工作。这不是答非所问。

但有三个值得追问的地方：

**第一，方案的具体性远超"可见性"的范畴。** "Lina项目费28万（70万的40%）""第一周进行访谈""第三周制作可交互系统地图""使用Observable或D3.js"——这些是项目执行细节，不是"让参与者看见彼此价值"。基因组说"协议不生产行为，协议生产可见性。行为是可见性的副产品，由参与者自主产生。"但这份方案就是在生产行为。

**第二，方案是系统生成的还是参与者涌现的？** SIM-002的评估报告指出，v1方案中"端侧自主涌现的内容比例大幅提升"。信号系统、每日粗剪同步、版权分层确实来自端侧自发提出。但方案的整体框架——三个月分阶段、联合服务包模式、具体定价——仍然是方案生成器（一个独立的LLM调用）综合产出的。参与者并没有"协商定价"或"讨论时间表"，这些是系统替他们做的。

**第三，如果参与者不认可方案怎么办？** 这个问题在实验中无法回答（所有角色都是LLM扮演的）。但在真实场景中，这是核心问题。基因组说"行为决策权不可被协议剥夺"，那方案就只能是"建议"。如果只是建议，方案生成器的价值是什么？如果参与者可以完全忽略方案自己谈，那方案生成步骤对通爻的核心使命有什么不可替代的贡献？

### 3. "哲学洞察"的实用性

SIM-002发现了"观测者效应"——Lina在场观察会改变被观察的系统（"第一天就告诉CEO Lina在观察，真实状态还存在吗？"）。评估报告将其标注为"极高价值"，并认为质量超出了预设的ground truth。

**从产品角度，我要区分两件事：这个洞察对这次结晶有用吗？这个洞察对通爻有用吗？**

对这次结晶：有用。它揭示了三人协作中一个真实的操作矛盾。Lina的"隐形观察"定位和"被介绍为观察者"之间的冲突是一个需要解决的问题。催化把这个矛盾显性化了，端侧可以据此调整自己的策略。这正是"消除信息差"在做的事。

对通爻：无直接价值。这是一个场景层的洞察，不会影响协议的设计。它不会改变编码方式、场结构或匹配逻辑。它验证了催化Agent有深度思考能力，但通爻的核心竞争力不是"催化有多聪明"，而是"让意图以趋近于零的成本找到相关意图"。

**更尖锐的问题是：实验团队把这类洞察当作结晶的核心价值宣传，是不是在不自觉地把通爻从"相关性协议"拉向"智能咨询平台"？** 如果通爻的卖点变成"我们的催化Agent能产出哲学级别的深度洞察"，那通爻就在跟McKinsey竞争，而不是在做TCP/IP。基因组里的类比是TCP/IP，不是McKinsey。

这不是说哲学洞察没有价值。是说它的价值归属是催化Agent的prompt质量，不是通爻协议的设计。好的催化prompt是应用层的优势，不是协议层的功能。

### 4. 跨语义翻译的性质

v1催化产出了24条跨语义翻译，例如：
- "赵维说的'把模糊需求翻译成清晰系统设计'和Maya说的'让不同角色的人能听懂彼此'可能在描述同一种核心能力：跨领域翻译"
- "三人用不同语言表达了同一个原则——'怎么开始'不是程序性问题，而是关系定义的时刻"

**这是通爻最应该做的事情。** 这就是"让投影的摩擦趋近于零"在自然语言层面的实现。

模块一用向量在数学空间做翻译（"这两个向量距离近"），模块二用LLM在自然语言空间做翻译（"这两段话可能在说同一件事"）。两者是同一个操作（投影）在不同精度上的实例。模块一做粗粒度匹配（"你们相关"），模块二做细粒度匹配（"你们在这个具体维度上相关"）。

**跨语义翻译是"发现"，不是"协作"。** 它扩展了参与者对彼此的可见性，没有替参与者做任何决定。催化说的是"可能在说同一种能力"，而不是"你们应该合作"。这个措辞严格遵守了"协议止于可见性"的原则。

v1催化prompt在这个维度上的成功，恰恰证明了模块二的催化循环部分是基因组的自然延伸。问题不在催化循环，问题在催化循环之后的方案生成。

### 5. 阴阳循环

基因组要求"意图 -> 数据 -> 新意图"的循环，没有起点和终点。

结晶的6轮循环是不是这种循环？

**形式上是，实质上不完全是。**

每一轮：端侧表达（意图投影）-> 催化观察（数据）-> 端侧基于观察调整表达（新意图）。这确实是阴阳交替。v1的R1->R2新发现从9个降到3个，说明循环在收敛——信息差在被消除。

但基因组描述的循环有一个关键特征：**任何行为都产生新意图，循环不依赖于"拒绝"才发生。** 结晶的循环在收敛后就停止了。收敛之后，参与者拿到方案，做出行为（接受/拒绝/修改），这些行为应该产生新意图回流到场中。

设计文档提到了这个："残余张力回到模块一的场中，触发新一轮场发现。" 这是正确的设计。但在实验中，循环在方案产出后就结束了。没有"方案被拒绝后残余回流"的模拟，没有"方案被部分接受后产生新张力"的模拟。

**实验只验证了半个循环。** 阴->阳（意图->结晶->方案）走通了，阳->阴（方案执行->新意图->新匹配）没有被触及。这不是实验的缺陷（POC阶段可以理解），但如果模块二在产品层面不包含回流机制，它就是一个线性管道，不是阴阳循环。

具体建议：结晶的产出应该是**结构化的可见性信息**（谁和谁在什么维度上相关、什么信息差被消除了、什么残余未消解），而不是一份完整的行动方案。前者天然可以作为新意图回流到场中。后者是一个终端产物，回流的阻力大。

### 6. 自愿参与

设计文档说："模块一输出的邻近图"是结晶的输入。也就是说，模块一（场发现）选出了参与者。

**但谁决定参与？是系统选的还是参与者自己决定的？**

基因组说"协议不生产行为，协议生产可见性。行为是可见性的副产品，由参与者自主产生。" 如果系统自动把匹配到的人放进一个结晶房间，那就是系统在替参与者做行为决策。

在实验中，这个问题被回避了——三个参与者是实验者预设的，没有"要不要参与"的选择环节。

**真实场景中的正确流程应该是：**
1. 模块一产出匹配结果（可见性）
2. 匹配结果被呈现给触发者（CEO）和被匹配者（Lina/赵维/Maya）
3. 每个人自主决定是否参与结晶（行为）
4. 只有同意参与的人进入结晶

这里有一个产品设计的难题：如果被匹配者不知道自己为什么被匹配，他可能没有足够的信息来决定是否参与。**结晶的邀请本身需要包含足够的可见性**——不是"你被系统选中了"，而是"系统发现你的XX能力和另一位参与者的YY需求可能相关，是否愿意进入一个对话来探索这个关联？"

**结晶过程中参与者能否退出？** 设计文档没有明确说。方案中有退出机制（每阶段可退），但那是方案层面的退出，不是结晶循环层面的退出。在6轮循环中，如果一个参与者在第3轮觉得"这跟我无关"，他能退出吗？协议应该允许。"行为决策权不可被协议剥夺"意味着参与者在任何时刻都可以选择沉默或离开。

实验没有测试这种情况。在真实场景中，这很可能发生。

## 关键风险

**风险1：协议层与应用层的边界再次模糊。** 基因组v0.3批评V1的一个核心问题是"synthesize从协议层移到应用层"（SS8）。模块二的方案生成步骤正在重蹈覆辙。催化循环是协议层，方案生成是应用层。如果不在架构上明确分离，模块二会成为V1 Center Agent的翻版——一个越界做应用层事务的协议层组件。

**风险2：通爻的定位从"发现层协议"滑向"咨询服务平台"。** 如果核心卖点是"催化Agent能产出多深刻的洞察和多完整的方案"，通爻就在跟咨询公司竞争。而基因组的定位是TCP/IP——基础设施层。这两个定位的商业模型、竞争格局、规模化路径完全不同。

**风险3：度量偏离。** 基因组的度量是换手率（阴阳循环的转动次数）。结晶实验的评估指标是Ground Truth发现率和方案质量。这些是实验指标，但如果产品层面也用这些指标，就是在度量"单次结晶有多好"，而不是"场有多活跃"。通爻的价值不在于一次结晶多精彩，在于大量意图持续流动、持续匹配、持续产生新意图。

**风险4：自愿参与机制缺失。** 当前设计假设模块一选出的人会自动进入结晶。在真实场景中，这需要一个明确的邀请-同意流程。如果这个流程设计不好，用户会觉得被系统"拖入"一个对话。

## 建议

**建议1：在架构上明确分离催化循环和方案生成。** 催化循环（跨语义翻译、信息差消除）是模块二的核心，属于通爻协议层。方案生成是可选的应用层功能，不属于模块二。结晶的标准产出应该是**结构化的可见性报告**——发现了什么关联、消除了什么信息差、残余了什么张力——而不是行动方案。方案生成可以作为一个独立的应用层插件存在，由有需要的场景自行启用。

**建议2：重新定义结晶的"成功"。** 当前实验用"方案是否具体可执行"来评判成功。从基因组角度，成功应该是：参与者在结晶后对彼此的了解是否显著超过了模块一的匹配结果？信息差是否被有效消除？参与者是否自主产生了后续行为（而不是系统替他们规划的）？换手率是否因此提升？

**建议3：设计邀请-同意流程。** 在下一次实验中，模拟参与者收到结晶邀请后自主决定是否参与的环节。邀请信息应该包含足够的可见性（为什么你被匹配、与谁相关、在什么维度上），但不应该包含任何行动建议。

**建议4：测试回流机制。** 在下一次实验中，模拟结晶产出后的回流：方案被部分拒绝 -> 残余张力 -> 重新进入场 -> 新一轮匹配。只有验证了完整的阴阳循环，才能说模块二不是线性管道。

**建议5：警惕"催化智能"成为卖点。** 催化Agent的深度洞察确实令人印象深刻。但通爻的竞争壁垒不应该建立在"催化有多聪明"上——因为任何有LLM API access的人都能做一个催化Agent。通爻的壁垒在场（大量意图的持久索引和实时匹配）和编码（投影的摩擦趋近于零）。催化是锦上添花，不是核心护城河。

**建议6：跨语义翻译才是核心资产，独立提取。** v1催化的24条翻译是最有价值的产出。考虑把翻译能力从结晶循环中抽象出来，作为通爻协议层的通用能力——不只在结晶中用，也可以在模块一的匹配结果展示中用。"系统发现你和张三相关，因为你说的X和他说的Y可能在描述同一件事"——这比纯粹的相似度分数有用得多，而且严格属于可见性。
