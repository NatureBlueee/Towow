## 第一部分：用户校准记录

以下是用户在审查 EXP-009 过程中给出的批评和校准，按主题归类。这些是硬性约束，后续所有复盘和实验迭代必须遵守。

### A. 审查方法论错误

**A1. 不展示原始内容就下结论**
> "一个一个都要我判断。但是你都没有给我看。说这个实验的过程是什么，这个是什么，那个是什么，我怎么给你提出评价？"

**校准**: 必须把实际内容展示给用户看，让用户自己判断催化工作得怎么样。不要自说自话地总结实验。

**A2. 用搜索范式指标评估响应范式**
之前的审查团队用 "30%→80% GT 发现率" 作为核心指标。但 GT 命中率是搜索范式（precision@K），无法衡量"意外发现"和"涌现价值"——这恰恰是通爻的核心。

**校准**: 评估标准 = 外部客观判断。作为客观外人：催化过程是否让 Agent 回复质量越来越好？最终方案是否有效消解了原始意图？

---

### B. 根本性认知错误：人类中心思维

**B1. 参与者 = Agent，不是人**
> "参与者始终是Agent，他不是人的代替，人不参与讨论...agent是世界的主体"

人不参与结晶讨论。Agent 带着丰富上下文（Profile/SecondMe/历史行为）代理人类在数字宇宙中交互。人只输入意图 + 接收执行通知。

**B2. 不存在异步问题**
> "所有这一切...全是发生在后台的后端的，用户最多是实时看到后端的日志记录而已"

Agent 随时可用，调用即回复。不存在"Maya 早上回复、赵维下午才看手机"这种场景。

**B3. 同一模型 + 不同 Profile = 生产架构，不是缺陷**
审查报告把"三个端侧都是同一个 Sonnet 模型"当致命缺陷。但 V2 期间端侧就是同一种大模型，个性化来自 Profile 数据。模型是透镜（基础设施），Profile 是投影内容。

**B4. 端侧数据假设为真实**
> "端侧你就假设它就是有数据的呀，就是会有真实的数据啊"

实验中端侧 Agent 基于 Profile 生成的数据（定价、时间表等），在生产环境中会是真实数据。不要批判"数字是编的"。

---

### C. 协议理解错误

**C1. 协作构型 = 张力消解**
> "分工本身能不能有效地消除信息差...不是去说什么'执行了才发生真实的消解张力'"

Agent 形成协作构型 → 通知人类用户 → 人类执行。方案本身就是消解，不是"执行后才消解"。

**C2. 执行后 = 新数据，不是残余张力回流**
> "获得新的数据...不仅是他们自己的 profile 有更新了，还有很多东西更新了"

执行后产生的是新数据回流到场（profile 更新等），不是"残余张力回流"。

**C3. 方案生成 ≠ 越界**
ADR-014 §4 明确：催化循环过程中"协议止于可见性"约束催化行为；收敛后方案生成是协议的自然终止态，属应用层。

---

### D. 实验目标错误

**D1. 不是验证可行性，是找影响因素**
> "不是验证它的可行性，而是去看怎么样才能更好地做到。我们就假设它都能做到的，我们就看什么因素在影响效果，然后一个一个去改"

实验目标 = 假设能做到，看什么因素在影响效果，一个一个改，改出一个方向。

**D2. 不要自嗨**
> "绝对不是说...因为你做了实验，你觉得很好，所以很好"

要回溯到基因组和设计文档，看实验是否真正服务于目标。

**D3. 哲学洞察的评价标准**
> "如果这个哲学洞察本身是那个提出意图的那个人需要的...那就非常非常好。但是如果不需要呢？"

催化的每一句输出都应该服务于"消除与张力消解相关的信息差"，不是生产知识。

**D4. 模块二的成功标准**
> "有没有解决张力，有没有解决那个最开始的意图的张力"

模块二的评估标准不是模块一的（让意图找到相关意图），而是：最终方案是否有效消解了触发结晶的原始张力。

---

### E. 上一次审查中确认的技术问题（有效批评）

这些是审查中提出的、经过校准后仍然成立的问题：

1. **翻译后期退化**: R1-R2 是真正的跨语义翻译，R4-R6 退化为同域操作细节对齐
2. **翻译固定 4 条/轮**: 模式锚定（satisficing），不是真实发现
3. **收敛完全失效**: max_tokens=3000 截断 + 催化无法区分真实信息差和讨论副产品
4. **五级优先级未体现**: 催化平铺输出，没有按收敛影响力排序
5. **方案中的"共识"不是真实共识**: 赵维单方提议被方案生成器当共识（催化在 R3 未执行优先级第 4 条）
6. **催化不知道翻译任务已完成**: 不存在"相变检测"机制
7. **端侧太被动**: 催化填补了端侧该做但没做的事

## 第二部分：转向决策（2026-02-19）

### AI 模拟复盘 → 真人实验

**决策**：放弃逐轮复盘 SIM-002，转向用真人数据跑新实验。

**理由**：
1. AI 模拟的根本局限：同一模型扮演三个角色再扮演催化 = 系统在自言自语。语义鸿沟是制造的，不是真实的
2. 已有 ~10+ 真人 Profile 和 ~10+ 真实需求，足以构成实验池
3. AI 模拟已完成历史使命：把 prompt 从完全不工作的 v0 调到基本能工作的 v1
4. 真人数据的评估更诚实：需求方本人可以判断"这有没有帮到我"

**保留的价值**：
- 第一部分的校准记录（A-E 类）全部适用于真人实验
- E 类技术问题（翻译退化、收敛失效等）是 prompt 级别的，真人数据上大概率同样存在
- catalyst_v1 作为起点，在真人数据上先看真实表现再决定改什么

**新实验指导文档**：`docs/research/014-real-human-experiment-guide.md`
