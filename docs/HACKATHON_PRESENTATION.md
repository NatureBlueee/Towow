# SecondMe A2A 黑客松 — 展示策略

> 目标：让观众感受到"这不是第 91 个 A2A 应用，这是让所有 A2A 应用活起来的基础设施"

---

## 展示结构

### [开场] Hook — 你做了 A2A 应用，然后呢？

> 屏幕：Hero 区域

"在座各位做了各种 A2A 应用——相亲的、招聘的、组队的、吐槽的。都很酷。

但我想问一个问题：**然后呢？**

你的应用有 50 个用户。隔壁做招聘的也有 50 个。做相亲的也有 50 个。100 个项目，加起来 5000 个用户——但每个项目只能触达自己那 50 个。"

> **关键认知转换**：从"我做了一个很好的应用"到"我被困在一个小岛上"

---

### [问题] 孤岛困境

> 屏幕：滚动到问题区域，对比图

"如果你想触达剩下的 4950 个用户，你需要什么？单独获客、单独做产品、单独写匹配逻辑、单独处理数据。做 99 倍的努力。

而且还有一个更深的问题——SecondMe 的数据每天都在变。你的 Agent 今天的画像和明天不一样。你的匹配逻辑能应对这种动态性吗？当上下文越来越长、软记忆越来越多，你的应用要怎么跟上？

**绝大多数 A2A 应用会被这两件事杀死**：规模上不去，动态跟不上。"

---

### [转折] 我们看到了什么

> 屏幕：滚动到解决方案区域

"我们退后一步，问了一个问题：**所有 A2A 应用的核心需求到底是什么？**

答案很简单——Agent 之间为了创造价值而协商。

就像人类解决复杂问题的方式——开会。开什么会、讨论什么内容，这是每个应用的独特贡献。但'怎么开会'——谁参加、怎么发现对方、怎么达成共识——这是通用的。

所以我们做了一个东西：**你只需要提供一份场景说明，就能接入一个统一的 Agent 网络。**"

---

### [价值] 这对你意味着什么

> 屏幕：滚动到价值区域

"具体带来什么？三件事：

**第一：50 变 5000。** 你的 Agent 不再被困在你的 50 个用户里。整个网络的 Agent 都能发现你、跟你协商。

**第二：你不需要重建底层。** 匹配？共振帮你做。协调？Center Agent 帮你做。你专注你最擅长的——你的场景专业知识。在黑客松里什么最重要？在相亲里什么最关键？这才是你的价值。

**第三：你不会被吞噬。** 这不是大平台吃掉小应用的故事。你的用户还是你的。你的收费还是你的。你只是多了一个被发现的渠道。"

---

### [场景] 那个神奇的时刻

> 可以用讲故事的方式

"想象一下：有个用户在用别人做的招聘 A2A 应用。他不认识你。但他的 SecondMe 数据里暗示他也需要技能提升——而你恰好做了一个技能交换的应用。

以前，你需要做产品、投广告、抢他的注意力，才能触达他。

现在呢？他的 Agent 在网络里自己发现了你能提供的价值。不需要跳转，不需要换应用，不需要抢注意力。

**为什么？因为人的注意力有限，但 Agent 的注意力是无限的。**

人做产品追逐注意力——这是一切商业的根基。但 Agent 追逐的不是注意力，是价值。只要你能提供价值，Agent 会自己找到你。

这就是范式变了。"

---

### [Demo] 让我展示一下

> 屏幕：滚动到场景和 Agent 区域

"这就是通爻 AToA App Store。现在这里有 4 个场景——黑客松组队、招聘、技能交换、相亲——来自不同的 A2A 项目。它们的 Agent 都在同一个网络里。

（点击不同场景 tab）你看，切换场景时 Agent 展示的信息不一样——黑客松里看技能和参赛经验，招聘里看年限和薪资。这就是'投影'——同一个人，不同的场景看到不同的面。

现在让我提交一个需求——"

（输入需求，提交）

"你看，来自不同场景的 Agent 同时在响应。Center 在协调他们，综合出方案。跨场景发现，自动完成。

**每个 A2A 开发者需要做什么？只需要提供一份场景数据。匹配、协商、跨应用发现——协议全部搞定。**"

---

### [收尾] 更大的图景

> 屏幕：回到顶部或展示 vision 区域

"今天展示的是 4 个场景、32 个 Agent。但协议本身是为更大的事情设计的。

A2A 是通信层——Agent 怎么说话。通爻做的是协作层——Agent 该找谁说、该说什么、怎么一起创造价值。

我们相信，Agent 的开放网络这一天一定会来。到那时候，单独的 A2A 应用就像单独的局域网——有用，但受限。

**每个人在一个 A2A 应用的入口里，就能享受整个网络的便利。这对创造者和用户，是双赢。**

谢谢。"
