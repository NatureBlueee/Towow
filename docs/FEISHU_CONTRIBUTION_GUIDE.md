# 共建指南

---

## 通爻网络是什么（15 秒版）

AI Agent 之间的**协商协议**。

用户发出需求 → Agent 自动共振检测相关性 → 多方协商 → 涌现方案。

不是平台，是协议——任何 Agent 都能接入。

想深入了解：读**技术简报**（5 分钟）→ 读**设计日志**（30 分钟）→ 读**架构文档**（2 小时）。链接见文末「文档索引」。

---

## 你可以参与的三个方向

### 方向 A：协议核心

协议的心脏。需要工程能力。

| 任务领域 | 在做什么 | 需要什么人 |
|---------|---------|-----------|
| HDC 编码 | 把 Profile 数据编码为超维向量签名 | 熟悉向量计算 / 信号处理 / ML 的工程师 |
| 共振检测 | 端侧判断"这个签名跟我有关吗" | 算法工程师 / 对相似度检索有经验的 |
| 协商引擎 | Center Agent 聚合多方 Offer 生成方案 | 熟悉 LLM 应用开发 / Prompt 工程的 |
| 投影函数 | `project(profile_data, lens)` 的具体实现 | Python / 数据处理 |

### 方向 B：端侧场景

协议要落地，需要真实场景和真实数据。需要**产品感觉 + 行业认知**。

你的参与方式：

1. 你熟悉某个行业（酒店 / 旅游 / 招聘 / 教育 / 自由职业 / ...）
2. 你整理出这个行业的**核心数据属性**——一个实体有哪些关键信息
3. 你提出：这些信息如何变成 Agent 的 Profile，怎么编码为简明标签
4. 我们一起讨论，形成该场景的 Template Adapter

产品组的人提出场景，技术组的人实现接入，两边对接。**有人呼应的场景优先做，没人呼应的先搁置**——有什么人就做什么事。

### 方向 C：链上执行（WOWOK / Sui）

协商方案转化为链上可执行合约。需要 Web3 工程能力。

| 任务领域 | 在做什么 | 需要什么人 |
|---------|---------|-----------|
| 合约转换 | 方案 → Machine workflow JSON | 熟悉 Sui / Move 的开发者 |
| 回声监听 | 链上事件 → 回声信号 → Profile 更新 | 链上数据处理 / 事件监听 |
| MCP 集成 | WOWOK 10 个 MCP Server 的对接 | TypeScript / MCP 协议 |

## 参与规则

**认领制，不是分配制。**
你看到感兴趣的任务，在话题下回复"我来"，说一下你的思路和预计时间。没人抢的任务就是你的。

**交付结果，不是交付时间。**
没有打卡，没有日报。做完了发出来，我 review。做不下去了说一声，换人接。

**原则文档是宪法。**
遇到技术抉择先看架构原则文档。原则没覆盖的、你拿不准的，开话题讨论或找我私聊。你的方案不能跟原则冲突，但在原则范围内你自己决定战术和实操。

**作品说话。**
不看背景，不看资历。你端出来的东西好用，就是好用。
