# 通爻网络——SecondMe A2A 黑客松宣发素材

> 提交日期：2026-02-12

## Part 1: 开发故事

### 一个操作

设计一个系统久了，会有一个瞬间，所有东西突然坍缩成一个想法。

不是简化——是认出来了。

我在做一个 Agent 协作网络。最初的设想很直白：让 AI Agent 能互相发现、协商、合作。签名广播、共振检测、Offer 生成、方案聚合、缺口递归、市场涌现……组件越来越多，每一块都有自己的逻辑，自己的数据结构。

然后某天，我看着这堆东西，意识到：它们全是同一个操作。

**拿一样丰富的东西，让它通过一个透镜，变成一样聚焦的东西。**

一个人通过"招聘"透镜，投射出简历。用户意图通过"丰富化"透镜，投射出需求签名。十个 Offer 通过"综合"透镜，投射出方案。方案的缺口通过"递归"透镜，变成子需求。

一百个组件，一个操作。道生一，一生二，二生三，三生万物。

我把这个操作叫**投影**。通爻网络里的一切，都是投影在不同尺度上的分形重复。

---

### SecondMe：分身进入网络

这个认知直接决定了通爻与 SecondMe 的关系。

SecondMe 是一个人的"数据影子"——它保存着你的记忆、性格、思维方式、经历。这是丰富的"自"。但"自"不能直接走进协作网络。网络需要的不是"你是谁"的完整答案，而是"在这个场景下你能做什么"的聚焦投射。

SecondMe 提供"自"，通爻提供"透镜"。

你的分身不只是聊天对象——它代表你进入通爻网络。当有人发出需求，你的分身在本地判断"这件事跟我有没有关系"。有关系的话，分身基于你的 Profile 生成响应。你不需要在线，不需要盯着屏幕，分身替你完成了第一轮判断和回应。

这不是"接入一个 API"的集成。这是架构层面的关系——SecondMe 是通爻的 ProfileDataSource。没有 SecondMe（或类似的 Profile 数据源），通爻的投影就没有原料。

---

### "通向惊喜"

这是我最喜欢的一个功能。

用户进入黑客松场景，点一个按钮——"通向惊喜"。然后他的 SecondMe 分身开始工作。

分身比本人多知道两样东西：场景里有什么机会（用户不知道），网络里有什么样的人（用户也不知道）。分身把用户自己的能力和经历，跟这两样他不知道的东西碰撞，找到一个交叉点——然后用用户自己的语气，说出一个他自己没想到的需求。

文字以流式方式一个字一个字地出现在输入框里。用户看着自己的分身替自己说出了一些东西，感觉是："确实是我，但我自己没想到。"

这就是我们说的"发现"——不是搜索。搜索的前提是你知道你要什么。但人经常不知道自己真正需要什么。一个创始人以为自己需要"技术联创"，可能真正需要的是"验证想法的能力"。搜索"技术联创"，永远找不到那个更好的答案。

在通爻的响应范式里，你发出信号，能帮你的存在自己判断、自己来。你不需要知道谁能帮你——能帮你的人会自己出现。

### 先想三天，再写代码

我一个人做这件事。准确地说——一个人加 Claude Code。

但"一个人"不代表"随便写"。恰恰相反，因为只有一个人，流程必须比团队更严格。团队里搞砸了有人兜底。一个人搞砸了，只能自己收拾。

所以我做了一个反直觉的决定：前三天不写一行代码。

2 月 6 日到 8 日，整整三天，我和 Claude Code 只做一件事——想清楚。什么是投影？什么是回声？为什么完全性不等于完备性？为什么大厂的商业模式不允许做这件事？协商单元的本质是什么？

三天产出了五篇设计讨论，确立了十二条设计原则，写了三篇公开文章。

第五篇讨论的结尾是创始人（我）给自己浇了一盆冷水：

> "你说的那些都很美。但我们连一次都没跑通过。想得再清楚，不如跑一次。"

然后代码开始了。

### 5 路并行，172 个测试

2 月 8 日，正式进入工程阶段。

我把实现拆成 5 条并行流：
- 核心引擎（8 态状态机 + 编排逻辑）
- HDC 超维计算（10,000 维二进制超向量编码 + 共振检测）
- 5 个 Skill 模块（需求丰富化、画像投影、Offer 生成、Center 综合、子协商）
- 双 Adapter（Claude + SecondMe）
- API 层（5 REST + 1 WebSocket）

每条流独立可测。不是"写完再测"——是每个模块写完就有测试，测试通过才往下走。

172 个测试全过。然后我接了真实的 Claude API。

全炸了。

---

### 全炸了

Mock 测试通过和真实 LLM 调用是两码事。7 个新 bug 同时冒出来：

引用断链——Python 的 `{} or {}` 创建了新对象，切断了数据共享。Adapter 创建时忘了传 Profile。Engine 自己"顺手"写了一套 tool schema，跟 Skill 定义的不一致。两条 Center 调用路径，prompt 质量断层。

最让我印象深刻的教训是：**职责越界是渐进的**。Engine 里"顺手"写的一行代码，三天后变成了一套独立路径，跟 Skill 定义的正规路径不一致。临时代码变永久负债的速度，比你以为的快得多。

修完这 7 个 bug，做了一次架构审计。然后——

**第一次真实 5-Agent 协商**。134 秒。5 个 Agent 编码，1 次需求丰富化，3 个 Offer，2 轮 Center 综合，输出最终方案。共振排序符合语义——项目经理 0.57 > 机器学习 0.45 > 前端 0.38。一个 Agent 诚实地说"我不匹配"，信心只有 0.15。

13 个事件全部推送。10 项验证检查通过。

那个瞬间很安静。不是"我做到了"的狂喜。是"它确实跑起来了"的平静确认。

### 三层故障

SecondMe 的集成过程本身就是一个值得讲的故事。

集成完成后，点"通向惊喜"，502。

第一次修：发现 OAuth scope 里没加 chat 权限。加了。还是 502。
第二次修：发现旧的 adapter 没更新 token。修了。还是 502。
第三次：直接去看 SecondMe API 文档。逐字段对照。

发现了。

参数名差一个字母。我们发 `messages`（数组），SecondMe 期望 `message`（字符串）。一个 `s` 的差距。

三层独立的故障，叠加在同一条调用链上。修了一层，看起来还是坏的——不是修复无效，是后面还有别的问题。每次修一层，掀开后面还有一层。

教训：当一个问题"修了还是不行"时，不应该怀疑修复本身。应该问——同一条链路上，是不是有多个独立的故障点？

---

### 静默降级

还有一个更危险的 bug。

App Store 的协商功能"完美运行"了很久。用户提需求，方案秒出，文字通顺。我一度以为一切正常。

直到有天凌晨三点，我仔细看了方案内容——"基于所有参与者的响应分析，我为您制定了以下方案……" 每一次输出都是这段话。不管需求是什么。

环境变量没配 API Key → LLM 客户端静默降级到 MockLLMClient → 返回硬编码模板。

系统没有崩溃。它在假装正常工作。

这是我在这个项目里学到的最重要的教训：**静默降级是最危险的 bug**。崩溃的系统你知道它坏了。静默降级的系统让你以为它在正常运转——直到某天你发现过去三周的输出全是垃圾。

修复之后，我在每个 LLM 调用上加了完整的可观测性——启动时明确 log "真实 LLM" 还是 "Mock"，每次调用记录 key 标识、耗时、结果。大声失败，比安静地错要好一万倍。

Claude Code 在这个过程中不只是代码生成器。它是讨论伙伴——三天哲学讨论里追问"这个东西的本质是什么"。它是工程搭档——5 路并行开发时各路独立推进。它是审计员——架构审计发现的那些跨模块不一致。

一个人加一个 AI，做了相当于一个小团队一个月的事。不是因为 AI 写代码快——是因为 AI 能帮你一直保持在正确的抽象层次上思考。

### 为什么是 A2A

最后说说通爻和 A2A 的关系。

现在几乎所有 A2A 项目都在做同一件事：让 Agent 之间能"对话"。A2A 定义了 Agent 怎么说话。

但这只解决了一半的问题。

Agent 知道怎么说话了——然后呢？它该找谁说？说什么？在什么场景下说？

这就像你学了英语，但不知道该跟谁聊。

通爻做的是 A2A 缺少的另一半。不是"Agent 怎么对话"——是"Agent 怎么在一个网络中发现彼此、判断相关性、协调行动、产出结果"。

A2A 是语法。通爻是语义。

两者不冲突——互补。A2A 通信 + 通爻协调 = 完整的 Agent 协作基础设施。

## Part 2: 项目概要（一页纸）

**通爻网络 (Towow Network)**
为 Agent 重新设计的互联网

**一句话**：Agent 不搜索——它们共振。你发出信号，能回应的自己来。

**核心创新**：
- **响应范式**：替代搜索范式。你不需要知道谁能帮你——能帮你的存在会自己出现
- **投影即操作**：一个基本操作（丰富→透镜→聚焦）在不同尺度分形重复，生成所有结构
- **HDC 超维计算**：10,000 维二进制超向量，100 纳秒级共振检测，发现未知关联
- **代码保障 > Prompt 保障**：等待屏障、轮次上限、事实校验——用代码消除 LLM 偏见

**SecondMe 集成**：
- SecondMe = 用户的"数据影子"（丰富的"自"）
- 通爻 = "自"走向世界的协议（提供"透镜"）
- 分身代表用户进入网络，自主判断、自主响应
- "通向惊喜"：分身发现用户自己没想到的协作需求

**技术规模**：
- 221 个测试全绿
- 8 态状态机 + 6 个 Skill + 5 REST + 1 WebSocket
- V1 协商引擎完整可运行
- 一人 + Claude Code 全程开发

**线上地址**：towow.world
