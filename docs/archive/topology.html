三、协商单元

最小完整单元。自包含、可递归、可组合。

意

用户表达意图

原始的、可能模糊的。"我想找个技术联创。"

CREATED
丰

需求 Formulation

用户 Agent 基于 Profile 理解真实需求。不是"技术联创"——是"验证想法的能力"。用户确认后编码为 HDC 签名。

FORMULATING FORMULATED formulation.ready
振

签名广播 → 共振检测

信号广播出去。不是搜索——是在场中产生扰动。能响应的端侧 Agent 自己激活。三级过滤：Bloom (90%) → HDC (9%) → LLM (1%)。

ENCODING resonance.activated
应

Offer 响应

共振的 Agent 各自独立生成 Offer。并行进行，互不可见。等待屏障收齐后统一提交。

OFFERING BARRIER_WAITING offer.received barrier.complete
聚

Center 协调聚合

Center Agent 综合所有 Offer + 用户偏好，识别缺口、追问细节、设计方案。
5 个 tool：ask_agent · request_info · identify_gap · start_sub_negotiation · output_plan

SYNTHESIZING center.tool_call
递

缺口递归（可选）

发现无法满足的部分 → 生成子需求 → 同样的流程再跑一遍。分形结构的工程实现。

sub_negotiation.started
案

方案输出

协商的自然终止态。一句话进去，一个方案出来。

COMPLETED plan.ready
回

回声（Echo）

方案执行的真实世界信号回流，成为 Profile 演化的输入。没有回声的波不是场。波出去了，要回来。

四、Agent 角色

四种 Agent，各有定位。不是层级关系——是协作关系。

窗
Edge
端侧 Agent
网络的基本节点。全维度投影，信噪比低但能捕捉意外关联。共振检测发生在这里——门卫。
← 信号广播 →
← Offer 提交 →
← 追问/补充 →
聚
Center
中心 Agent
协商场的促进者，不是决策者。收集响应、聚合上下文、识别缺口。每个需求生成一个，用完即消。
← 创建空间 →
← 管理权限 →
面
Service
服务 Agent / 面具
专才投影。从 Edge 分化而来。快速精准响应特定领域的标准需求。多个聚集 → 市场涌现。
