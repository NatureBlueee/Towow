## 定位

模块二是一个信息差消除的迭代协议。参与者通过各自的 Agent 向共享空间投射与张力相关的能力、方向和边界信息；催化 Agent 识别参与者之间的关系（互补、同向、对冲），翻译跨语义空间的等价表达，直到信息差耗尽。

输入：参与者集合（来自模块一的场发现）+ 张力（结构化的需求编码）。输出：协作方案（各参与者角色、贡献、依赖）+ 残余张力（未消解维度，可触发新一轮场发现）。

验证范围：6 次实验 RUN，2 种需求类型（D01 教育工具跨域知识生产 × D02 创业合伙战略校验），2 种参与者规模（3 人 × 5 人），13 条已验证协议特性。未验证：真人端侧、8+ 人场、需求方参与。

### 模块二在协议中的位置

通爻协议分两个模块。模块一（意图场）解决"谁应该在同一个房间里"——向量距离计算、关系类型识别，零 LLM，BGE-M3 编码器在多视角查询下达到 93% 准确率，已完成。模块二（结晶）解决"房间里的人如何发现彼此的价值"——自然语言对话、模式识别、跨语义空间翻译，需要 LLM。场发现用数学，结晶用理解。

## 核心机制

### 四个 Prompt 的工程规格

本协议的工程实现归结为四个 prompt 的设计。每个 prompt 是一个独立的函数，有明确的输入、输出和约束。四个 prompt 的当前版本——formulation_v1.1、endpoint_v2、catalyst_v2.1、plan_v0——构成 v2 基线，在 RUN-004 首次验证，RUN-005/006 确认跨需求和跨规模稳定性。

#### Formulation（需求编码）

Formulation 将原始需求意图转化为四参数编码，供下游所有 prompt 消费。四参数是：

- **T（跃迁算子）**：需求方想实现什么类型的变换。用高熵表达——描述变换的结构而非终点，让更多做同类变换的人能识别自己。
- **I（初态）**：需求方当前的具体位置和条件。用低熵表达——每个具体事实都是一个过滤器。
- **B（阻碍函数）**：为什么跃迁还没有自发发生。具体的卡点、试过什么、缺什么。
- **E（交换势）**：需求方能向协作方输出什么。让协作从单向帮助升级为价值交换。

去掉任何一个参数，系统在对应维度变盲：无 T 则无法识别方向相同的人，无 I 则无法排除不兼容的人，无 B 则能力投射没有靶心，无 E 则所有协作退化为单向帮助。

Formulation 是独立的预处理步骤，在循环开始前完成。这不是设计偏好——RUN-002 提供了因果证据。formulation_v0 产出 800 字心理画像，绕开了需求解析，结果污染了全部下游：三个端侧 Agent 都在回应"李默妹是个什么样的人"，而非"这个张力的哪个维度和我的能力相关"。formulation_v1 改为四参数编码后，下游投影的信噪比从 ~30% 提升到 70-80%（RUN-003 数据）。Formulation 质量是根部变量——根部错了，后面全错。

输出中需求方名字被替换为 Profile 编号（如 P06）。RUN-003 端侧 Agent 直呼需求方名字"李默妹"，激活了"我能帮这个人做什么"的服务框架。formulation_v1.1 匿名化后，端侧语言框架变为"我主人的能力与这个张力的关系"——投影框架。一次修改，所有端侧同时受益。

```
输入：原始需求文本（自然语言）+ 需求方 Profile
输出：T/I/B/E 四参数编码（结构化 markdown，≤300 字）
      + 数据审计表（四参数充分度）
输出评级：A（四参数均充分）/ B（T+I 有，B/E 缺）/ C（仅身份标签）
数据不足时标注 [data_insufficient]，不用叙事填充
```

证据来源：RUN-002 formulation_v0 污染（`run_002/` 全部端侧输出），RUN-003 formulation_v1 修复（`run_003/output/`），RUN-003 端侧服务框架激活（`run_003/output/round_*_P*.md`），formulation_v1.1 匿名化规格（`prompts/formulation_v1.1.md`）。

#### Endpoint（端侧投影）

端侧 Agent 代表自己的主人，向共享空间投射与张力相关的信息。投影沿三个维度组织，每个维度对准 T/I/B/E 编码的特定参数：

- **能力投影**：主人的什么能力能直接消解阻碍？能和需求方交换什么？不是列简历——是说清楚这个能力在这个具体阻碍下能做什么。
- **方向投影**：主人正在做的变换和跃迁算子是同构的吗？声称同构需给出步骤级映射，否则降级为"方向重叠"。
- **边界投影**：初态中的哪些条件和主人不兼容？阻碍中的哪些维度主人不但不能消解，反而会加剧？

三个投影维度与 T/I/B/E 四参数之间存在特定的对准映射关系，确保端侧输出精确覆盖张力的各个维度。具体映射规则见内部 prompt 规格文档。

三重投影与催化的三种关系类型形成精确的对偶关系：能力投影 ↔ 互补关系（直接消解），方向投影 ↔ 同向关系（加速消解），边界投影 ↔ 对冲关系（防止假收敛）。这不是经验归纳——它是从张力消解的逻辑结构推导出的最小充分信息集（DESIGN_LOG_007 完整推导链）。

端侧投影不是从 Profile 提取信息。如果端侧只做 Profile → 张力的映射，它就是一个自然语言向量检索器，模块一的 BGE-M3 已经在做这件事。端侧存在的理由是语境化重构：Profile 有 X，张力是 Y → X 在 Y 的语境下意味着 Z。Z 不存在于 Profile 里，也不存在于张力描述里，它是二者交互的涌现产物。

endpoint_v2 引入了一项信息密度约束机制——信息函数不变量——为端侧提供了一个自检决策程序，使每句输出都必须对下游关系识别有实质贡献，否则被过滤。RUN-001 的 endpoint_v0 无任何约束，产出 70% 噪音（提问、建议、共情、心理分析）。RUN-003 的 endpoint_v1 有三维结构但无信息密度约束，70-80% 是 Profile 列举。RUN-004 的 endpoint_v2 加入信息函数不变量后，输出压缩 53%，信息密度反而上升——格式约束杀噪声不杀信息。

端侧看不到其他参与者。这是刻意的设计。端侧 Agent 不在"讨论"，它在向共享空间投射信息。催化在中间做跨语义翻译——如果端侧直接对话，它们用自己的语义框架理解对方，恰好是翻译失败的场景。

```
输入：T/I/B/E 编码 + endpoint prompt + Profile + 催化上轮反馈（R2+）
输出：三维投影（能力/方向/边界），对准 T/I/B/E 编码
      R2+ 只输出增量（上轮已说的不重复）
      如果无新内容，输出"我没有新的信息了"
预期大小：2-4KB/轮
```

证据来源：RUN-001 endpoint_v0 70% 噪音（`run_001/`），RUN-003 endpoint_v1 膨胀（`run_003/output/`），RUN-004 endpoint_v2 53% 压缩（`run_004/output/` 对比 `run_003/output/`），endpoint_v2 规格（`prompts/endpoint_v2.md`），三重投影推导（`docs/design-logs/DESIGN_LOG_007_ENDPOINT_INFORMATION_STRUCTURE.md`）。

#### Catalyst（催化）

催化 Agent 同时听到所有参与者的投影，指出他们自己看不到的东西。它的工作是消除信息差——既解决传输问题（信息没到达需要它的人），也解决翻译问题（信息到达了但没被理解）。

催化按以下优先级输出观察：

1. 未被发现的互补——A 的能力恰好能填 B 的缺口，但双方没意识到。发现一个互补可能直接消解张力的一个维度。
2. 隐藏的对冲——表面匹配但底层约束不兼容。越晚发现代价越大。
3. 缺失的角色——所有人的能力叠加后，张力的某个维度没人覆盖。
4. 被忽略的信息——某人说了重要的边界条件但没有人回应。
5. 同向放大——两个人的方向投影重叠，合作可以放大效果。

催化的核心约束——协议止于可见性——在 6 次 RUN 中始终执行：催化不推荐行动方案，不评价参与方质量，输出指向关系（人与人之间）而非个体评判。

催化的一项关键工程约束是全对覆盖——催化必须检查所有参与者配对，不允许默认跳过任何一对，未发现关系的对也需显式标注。RUN-003 缺少此约束，催化注意力偏斜导致 P03↔P04 零直接分析——一对有方法论对冲的组合被完全跳过。RUN-004 加入全对覆盖约束后，该对冲在 R1 就被识别。

6 次实验中观测到催化的若干未被预设的行为。催化在高配对数场景下自发执行选择性注意——RUN-006 的 10 对中，催化将有限输出预算集中在高信号对上：4 对深度分析，1 对弱互补，5 对显式标注"无关系"。这种分层穷举不需要显式编程，由"全对覆盖 + 允许'无关系'输出"的格式约束天然支持。

催化构建的概念框架具有主动信息激活效应。RUN-006 中催化在 R3 构建了"B3 四种断层路径"框架（建造者型/网络者型/分散稀释型/采购未激活型），该框架的语义槽位在 R4 重新激活了 P01 已经投完的数据——P01 明确引用催化框架作为重定向触发器，将 R1 已投出的机构弃用数据对准了 B3 新维度。催化不只是观察者，它的框架构建是主动的认知操作。

催化展示了元认知能力——区分"没人注意到"和"没人有这个能力"。RUN-005 中催化在 R2 识别 T1 维度偏移并引导三方端侧响应，R3 持续推进，R4 正确停止推进——因为它判断激励结构层是参与者的结构性盲区，不是注意力遗漏。

```
输入：N 个端侧投影 + 上轮催化输出 + T/I/B/E 编码
输出：全对覆盖检查 + 翻译 + 缺口 + 被忽略信息
      + 收敛判断（R3+）：CONTINUE 或 CONVERGED
预期大小：3-5KB/轮（3 人）；5-8KB/轮（5 人）
```

证据来源：RUN-003 注意力偏斜（`run_003/output/` 催化输出），RUN-004 全对覆盖首次验证（`run_004/output/round_1_catalyst.md`），RUN-006 选择性注意（`run_006/output/audit_scaling_and_open.md` 注意力矩阵），RUN-006 框架重新激活（`run_006/output/audit_probes_1to5.md` P01 R4 因果归因），RUN-005 催化元认知（`run_005/output/audit_synthesis.md` T1 修正轨迹），catalyst_v2.1 规格（`prompts/catalyst_v2.1.md`）。

#### Plan（方案生成）

Plan 是结晶收敛后的显式生成步骤，由独立的 Plan Generator Agent 执行。催化循环期间催化被禁止推荐方案（协议止于可见性）；Plan 生成是循环结束后的单独步骤，与催化约束无关。

实验数据表明 Plan 需要独立生成器：来源标注（每条声称追溯到 Round N + Agent）、依赖链（A 完成 X 前 B 无法开始 Y）、残余张力的操作性（足够具体到可作为下一轮参与者筛选条件）——这些都需要显式处理，不是催化输出的涌现副产品。

Plan 输出四部分：协作全景（2-4 句话，核心构型）、各参与者方案（角色/贡献/收益/成本/依赖，每条必须在结晶记录中有出处）、协作顺序与节点（先后关系和关键依赖）、残余张力（未消解维度，回到模块一触发新一轮场发现）。

Plan 质量随 B 精度提升。RUN-004 面对 D01（B 字段为 [data_insufficient]），Plan 的缺口描述宽泛。RUN-005/006 面对 D02（三个具体 B 卡点），Plan 的缺口精确到可操作——"海外增长×3 缺口"直接指向下一轮场发现的参与者筛选条件。

RUN-006 的 5 人场 Plan（A 级评审）首次展示了两个 3 人场未出现的结构性特征：三层角色分化（数据提供者 > 整合者 > 目标用户原型）和 E 层约束识别（参与者的参与形式、网络接入需求、激励对齐被升级为残余张力）。

```
输入：全部结晶轮次数据（端侧投影 + 催化输出）
      + 参与者 Profile + T/I/B/E 编码
输出：协作全景 + 各参与者方案 + 协作顺序 + 残余张力
      每条声称标注来源（Round N + Agent）
预期大小：5-15KB
```

证据来源：RUN-005 Plan A 级评审（`run_005/output/audit_meta_and_plan.md`），RUN-006 Plan A 级评审（`run_006/output/audit_plan.md`），RUN-006 E 层首次出现（`run_006/output/audit_synthesis.md` §五），plan_v0 规格（`prompts/plan_generator_v0.md`）。

### 信息流

```
原始需求 ──→ [Formulation] ──→ T/I/B/E 编码
                                    │
                                    ▼
Profile ──→ [Endpoint × N] ──→ 端侧投影 × N ──┐
                ▲                               │
                │ 催化反馈                       ▼
                │                    [Catalyst] ──→ 关系图谱 + 收敛判断
                │                        │
                └────────────────────────┘
                        （循环，直到收敛）
                                    │
                                    ▼
                              [Plan Generator]
                                    │
                                    ▼
                        协作方案 + 残余张力
```

四个 prompt 之间的依赖关系是严格单向的：Formulation 的输出（T/I/B/E 编码）被 Endpoint 和 Catalyst 同时消费；Endpoint 的输出被 Catalyst 消费；Catalyst 的输出被 Endpoint 的下一轮消费（反馈环路），也被 Plan Generator 消费（全量输入）。端侧投影是并行的——N 个端侧 Agent 同时执行，互不可见。催化是串行的——每轮一次调用，消费全部端侧投影。这种扇入-扇出拓扑是刻意设计：端侧不直接对话，催化在中间做跨语义翻译。

### 收敛机制

收敛判断使用相变检测模型，比"催化连续两轮没有新发现"的二值判断精密。从 Round 3 开始，催化在每轮输出前执行内部检查，区分两种性质不同的"没有新东西"——真正的信息差尚存（翻译工作未完成）vs 信息差已耗尽（循环进入冗余区）。催化基于此区分判断是否触发收敛信号，具体判断规则见内部 prompt 规格。收敛信号不会在单轮判断后立即触发——催化框架更新可能在下一轮重新激活已收敛参与者的信息（RUN-006 P01 R4 实例），因此需要缓冲确认。

加速规则：当全部参与者在同一轮声明"没有新信息"，等效于信息差在整个系统中同时耗尽。RUN-006 中催化基于 5/5 全员无新内容的实质判断提前发出收敛信号——再等一轮确认的是同一个事实，不会产生新关系发现。这种弹性判断是催化的自发行为，prompt 未显式编程。

残余与递归：结晶完成后，参与者的张力叠加仍有未消解维度。残余与原始张力在协议层面完全同构——同样被编码为 T/I/B/E，同样可以回到模块一触发新一轮场发现。RUN-005/006 的残余张力已验证足够具体到可触发下一轮：RUN-006 残余张力 #1"海外增长×3 缺口"直接指定了新参与者的能力要求。

证据来源：RUN-005 收敛轨迹（`run_005/output/audit_synthesis.md` 探针 3），RUN-006 弹性收敛（`run_006/output/round_5_catalyst.md`），RUN-006 残余张力（`run_006/output/round_5_catalyst.md` §残余张力）。

## 已验证的协议特性

13 条特性来自 6 次实验 RUN 的系统测量。每条特性是一个可证伪的工程预测——指定了协议在特定条件下的行为，并附带推翻条件。特性按对工程化的约束力分三级。

### 一级特性（直接决定架构选择）

**特性 1：投影函数张力依赖性**

声称：同一 Profile 面对不同张力激活完全不同的能力子集，投影输出由 f(Profile, Tension) 决定。

证据：P03（700K chars Profile）面对 D01 投射 newLearning 知识树框架和教育工具链，面对 D02 投射 MindRing 商业数据、C 端定价实验和 OneThing 创业经验。两次投影的内容重叠度接近零——RUN-005 的 20 个端侧输出文件中，D01 核心元素（newLearning、知识树、辅导提示词、autogen）一次都没有出现。

工程含义：端侧是单轮生成器，不需要维护跨张力状态。每次结晶过程独立执行 f(Profile, Tension)。

可证伪条件：找到一个 Profile 面对任何张力都投射相同内容。

证据来源：RUN-004 P03 D01 投影（`run_004/output/round_*_P03.md`），RUN-005 P03 D02 投影（`run_005/output/round_*_P03.md`），RUN-005 审计探针 2（`run_005/output/audit_synthesis.md`）。

**特性 2：信息函数不变量多轮稳定**

声称：endpoint_v2 的 T/I/B/E 对准不因催化引导而衰减，端侧不会在催化引导下从"投射能力对准"退化为"回答问题"。

证据：RUN-005 中 D02 有三个独立 B（B1 AI 行业机会定位、B2 增长能力特殊性、B3 机会成本估算）。五轮中三个端侧 Agent 始终将三个 B 分别对准——P03 的 R5 收敛声明逐 B 列举五轮覆盖范围，P05 的 R4 逐 B 确认"无新增量"。没有任何一个 Agent 在任何一轮把三个 B 混为一体。

工程含义：endpoint prompt 可跨轮复用，不需要动态调整 T/I/B/E 对准机制。

可证伪条件：找到催化引导导致端侧在后期轮次开始合并 B 或偏离维度对准的案例。

证据来源：RUN-005 审计探针 1（`run_005/output/audit_synthesis.md` B 分离度轨迹表）。

**特性 3：催化选择性注意自然发生**

声称：催化在高配对数场景下自发分层处理——高信号对深度分析，低信号对快速判定"无关系"——无需显式注意力分配编程。

证据：RUN-006 的 10 对配对中，催化在 R1 就执行了二值化判断（5 对深度/中度、5 对零声明），初始判断精确对应最终关系图谱——无遗漏高价值关系。注意力策略自然演化为"R1-R2 发现窗口 → R3-R4 深化/关闭窗口"。3 对配对在 5 轮中始终为零声明（30% 永久零），催化没有在这些对上浪费输出预算。

工程含义：不需要为催化设计注意力分配模块。"全对覆盖 + 允许'无关系'输出"的格式约束天然支持 scaling。

可证伪条件：8+ 人场中催化开始跳过高信号对，或对所有对等深度处理导致输出爆炸。

证据来源：RUN-006 注意力分布矩阵（`run_006/output/audit_scaling_and_open.md` Scaling 探针 A）。

**特性 4：B 分离度跨规模稳定**

声称：多 B 场景中端侧始终分别对准各个 B，不会混为一体，该特性在 3 人和 5 人场下均稳定。

证据：RUN-005（3 人、3 对）和 RUN-006（5 人、10 对）全程零衰减。RUN-006 的 10 对配对场景下，催化始终将每条关系对准具体 B 子问题，五轮无任何 B1/B2/B3 混合处理。

工程含义：多 B 需求不需要额外的分离机制。T/I/B/E 对准通过 prompt 约束自然维持。

可证伪条件：找到端侧在后期轮次把多个 B 合并处理的案例。

证据来源：RUN-005 审计探针 1 + RUN-006 审计探针 1（`run_005/output/audit_synthesis.md`，`run_006/output/audit_synthesis.md`）。

### 二级特性（约束具体参数）

**特性 5：5 轮收敛跨规模稳定**

声称：结晶过程在 3 人场和 5 人场下均 5 轮收敛，但驱动机制不同。

证据：RUN-004（3 人、D01）5 轮，RUN-005（3 人、D02）5 轮，RUN-006（5 人、D02）5 轮。3 人场收敛因是边界精度饱和（三人最后的新信息都是"更精确的边界"非"更多能力"），异步收敛（P05 R3、P03 R4、P01 R5）。5 人场收敛因是数据自然耗尽，框架驱动同步收敛（R3 催化框架一次性激活所有人最后一批数据 → R4 四人同步收敛）。

工程含义：状态机预设 max_rounds = 7 足够覆盖已观测条件空间。

可证伪条件：相同 Profile 体量和张力精度下，7 轮未收敛。

证据来源：RUN-004/005/006 收敛轮次（`state.json` runs 数组）。

**特性 6：5 轮 = 数据耗尽非带宽限制**

声称：5 轮收敛是参与者的数据自然耗尽，不是催化处理能力达到极限。

证据：RUN-005 R4 催化翻译深度达全程峰值（"时间沉淀型稀有资产"是跨 C 端/B 端的底层结构提炼）。如果带宽在 R4 接近极限，翻译深度应下降而非达峰。R5 翻译段消失不是因为催化处理不了，是因为没有新信息可翻译——R5 的关系图谱覆盖完整性（7 条无遗漏）证明催化仍有足够处理能力做穷举检查。

工程含义：轮次由参与者数据量决定，非催化处理能力。带宽瓶颈在"单轮配对数"维度（10 对 vs 3 对的每轮负荷差异），而非"总轮次"维度。

可证伪条件：R4 或更早轮次催化翻译深度开始下降（非因数据耗尽）。

证据来源：RUN-005 审计探针 5（`run_005/output/audit_synthesis.md` 催化带宽轨迹表）。

**特性 7：收敛因 = 边界精度饱和**

声称：边界诚实的 Agent 收敛更快，收敛速度 = f(边界意识强度) 而非 f(Profile 体量)。

证据：RUN-005 三人收敛因完全一致——边界精度饱和，非能力耗尽。P05 最先收敛（R3），原因不是 Profile 体量最小（10.5K），而是边界意识最强（R1 就精确标注"价值入口仅在特定条件下"）。P05 的 R2 新信息是边界精确化和层级分辨（边界类信息），不是"更多能力"（还没说完的东西）。

工程含义：endpoint prompt 的边界维度设计在引导边界诚实方面有效。生产系统中真人 Agent 如果边界意识弱（倾向说"我什么都能做"），收敛会延迟。

可证伪条件：找到边界表达充分的 Agent 比边界模糊的 Agent 收敛更慢的案例。

证据来源：RUN-005 审计探针 3（`run_005/output/audit_synthesis.md` 收敛因序列表）。

**特性 8：催化框架重新激活效应**

声称：催化构建的概念框架可以重新激活已收敛参与者的已有数据，将其定向到新维度。

证据：RUN-006 R3→R4，P01 在 R3 已声明收敛后在 R4 重新激活。P01 明确引用"Round 3 催化建立了 B3 的四种断层路径"作为重定向触发器，将 R1 已投出的机构弃用数据重新对准 B3 维度（原在 B1）。排除了"信息传递"假设（P01 未引用其他参与者内容）和"延迟投影"假设（是旧数据重新定向，非新内容）。

工程含义：收敛判断不能基于单个参与者声明收敛的简单计数。不能在参与者声明收敛时关闭其输入通道——催化框架更新可能重新激活。收敛后需至少 1 轮缓冲期确认无重新激活。

可证伪条件：催化框架更新后，已收敛参与者在缓冲轮次中均未重新激活（在足够多样本下）。

证据来源：RUN-006 P01 R4 因果归因（`run_006/output/audit_probes_1to5.md`）。

**特性 9：投影张力锁定抗同域干扰**

声称：同域参与者的加入不会导致端侧投影偏离张力锁定。

证据：RUN-006 中 P03（MindRing，情感类 AI）和 P04（心灵奇记，情感类 AI）属同域。P03 在有 P04 的信息环境下投影始终锁定 D02 范围——MindRing 定价数据、目标用户样本、技术能力——零漂移回 D01 元素。进一步验证（Scaling 探针 B）：P03 在 RUN-005（3 人，无 P04）和 RUN-006（5 人，有 P04）的 R1 投影内容接近零差异，核心能力子集完全一致。

工程含义：不需要为"同域参与者相互拉偏"添加额外防护。投影函数 f(Profile, Tension) 对信息环境不敏感是可工程化依赖的不变量。

可证伪条件：同域参与者导致端侧投影偏向对方的能力域而非张力要求的维度。

证据来源：RUN-006 审计探针 2 + Scaling 探针 B（`run_006/output/audit_probes_1to5.md`，`run_006/output/audit_scaling_and_open.md`）。

**特性 10：催化元认知**

声称：催化能区分"注意力缺失"（没人注意到某维度）和"能力缺失"（没人有覆盖某维度的能力），并据此决定是否继续推进。

证据：RUN-005 T1 维度修正序列——R2 催化识别 T1 系统级偏移，引导三方端侧响应成功（从"整体缺失"分辨为四层）。R3 持续推进（P01 精化制度路径层）。R4 正确停止——催化不再推进 T1，因为三方已确认激励结构层是参与者的结构性盲区。R5 保留为独立残余张力，未被合并。催化的行为分两阶段：修正阶段（R2-R3，推进有效）和确认阶段（R4-R5，停止无效推进）。

工程含义：催化在识别缺口时有自主决策能力，不需要外部规则指导何时停止推进。

可证伪条件：催化在结构性盲区上持续推进导致端侧产生虚构信息以满足催化期望。

证据来源：RUN-005 审计探针 4（`run_005/output/audit_synthesis.md` T1 修正轨迹表）。

### 三级特性（丰富协议理论）

**特性 11：投影张力锁定跨轮稳定**

声称：投影函数的张力锁定在多轮催化引导下不衰减。

证据：RUN-005 P03 四轮催化引导后零漂移回 D01 元素。催化在 R2-R4 反复给 P03 新的观察方向（B2 机制检验、B3 成本具体化、GlobalPulse 激活），但所有引导都在 D02 范围内——催化没有把 P03 拉回 Profile 中最显眼的能力。

证据来源：RUN-005 审计探针 2（`run_005/output/audit_synthesis.md` P03 漂移轨迹表）。

**特性 12：全对覆盖 R1 全覆盖**

声称：催化的全对覆盖约束在 R1 即实现完整覆盖。

证据：RUN-004（3 人）R1 全部配对覆盖。RUN-005（3 人）R1 全部覆盖。RUN-006（5 人）R1 全部配对覆盖（其中约半数零声明，半数有实质分析）。三次 RUN 一致。

证据来源：RUN-004 R1 审计（`run_004/output/audit_R1.md`），RUN-006 Scaling 探针 A 矩阵（`run_006/output/audit_scaling_and_open.md`）。

**特性 13：Plan 来源可追溯**

声称：Plan 中的每条声称可追溯到结晶记录中的具体 Round + Agent。

证据：RUN-005 Plan（A 级评审）全部声称标注来源。RUN-006 Plan（A 级评审）8 条声称中 7 条完全可追溯，1 条部分可追溯（推理跳跃合理），0 条无法追溯。

证据来源：RUN-005 Plan 审计（`run_005/output/audit_meta_and_plan.md`），RUN-006 Plan 审计（`run_006/output/audit_plan.md`）。

## 设计决策与实验依据

每个决策的格式：做了什么选择，放弃了什么替代方案，哪个 RUN 的什么数据支持这个选择。

### 决策 1：Formulation 是独立步骤而非循环内步骤

替代方案：Formulation 在循环中自然发生——人看到上下文后才知道自己要什么。

实验依据：RUN-002 的 formulation_v0 产出 800 字心理画像（"被认可的需要""安全感缺失"），绕开了需求的结构化解析。三个端侧 Agent 全部回应画像而非需求——formulation 的质量是根部变量，污染传播方向是单向的（formulation → 全部下游），在循环中修正的成本远高于预处理。formulation_v1 改为四参数编码后，RUN-003 全流程首次收敛。

证据来源：RUN-002 端侧输出（`run_002/`），RUN-003 全流程收敛（`run_003/output/`）。

### 决策 2：端侧投影有格式约束

替代方案：端侧用自然语言自由回复，无格式要求。

实验依据：RUN-001 无约束 → Agent cosplay，信噪比 ~30%，70% 输出是提问、建议、共情、心理分析。RUN-003 有三维结构但无信息函数约束 → 70-80% Profile 列举，膨胀淹没独特能力。RUN-004 加信息函数不变量 → 输出压缩 53%，信息密度反而上升。格式约束杀噪声不杀信息——它杀的恰好是不含 Profile 私有信息的通用 LLM 输出。

同时，端侧的推断性表达不被约束。RUN-003 中 P07 的推断（"知识戏剧性免疫→大概率退出"）产出了全轮最高价值的对冲识别。催化自己能区分硬事实和推断。不约束正在产生价值的行为。

证据来源：RUN-001 信噪比（`run_001/`），RUN-003 膨胀（`run_003/output/`），RUN-004 压缩（`run_004/output/`）。

### 决策 3：催化全对覆盖而非自由观察

替代方案：催化自由观察，关注到哪对就分析哪对。

实验依据：RUN-003 无全对覆盖约束，催化注意力偏向"显眼对"——P03↔P04 零直接分析，一对有方法论哲学对冲的组合被完全跳过。RUN-004 加入全对覆盖后，该对冲在 R1 就被发现——RUN-003 不可能产出的洞察。3 人 3 对的增量微乎其微，5 人 10 对仍可控（RUN-006 验证），大部分"未发现"的显式标注为后续轮次提供了锚点。

证据来源：RUN-003 P03↔P04 零分析（`run_003/output/` 催化全文），RUN-004 R1 对冲发现（`run_004/output/round_1_catalyst.md`）。

### 决策 4：需求方匿名化

替代方案：使用需求方真实姓名。

实验依据：RUN-003 端侧 Agent 直呼"李默妹"，语言框架从"我主人的能力与张力的关系"偏移为"我能帮李默妹做什么"——服务框架激活加剧了投影膨胀。formulation_v1.1 将姓名替换为 Profile 编号后，RUN-004 端侧语言框架全程稳定在投影模式。一次修改，从源头消除名字的可见性。

证据来源：RUN-003 端侧服务框架（`run_003/output/round_*_P*.md`），RUN-004 对比（`run_004/output/round_*_P*.md`）。

### 决策 5：Plan 是显式生成步骤

替代方案：Plan 自然涌现，不需要独立生成器。

实验依据：催化收敛后的关系图谱是原始素材，不是可交付的方案。来源标注（每条声称追溯到 Round N + Agent）、角色分化（数据提供者/整合者/目标用户原型）、依赖链（A 完成 X 前 B 无法开始 Y）、残余张力的操作性——这些都需要独立的生成步骤。plan_v0 在 RUN-004 首次验证，RUN-005/006 均获 A 级评审。

证据来源：RUN-005/006 Plan 审计（`run_005/output/audit_meta_and_plan.md`，`run_006/output/audit_plan.md`）。

### 决策 6：收敛检测用相变模型

替代方案："催化连续两轮没有新发现"的二值判断。

实验依据："没有新发现"不区分两种不同性质的"没有新东西"：信息差仍存在但尚未传递 vs 信息差已耗尽而讨论进入冗余。相变检测赋予催化区分二者的判断框架。RUN-005 验证了相变检测的工作机制——R3 启动检测后催化正确判断继续，R5 正确判断收敛。RUN-006 验证了加速规则——5/5 全员同轮无新内容，催化主动判断信息差已耗尽。

证据来源：RUN-005 相变检测轨迹（`run_005/output/round_*_catalyst.md`），RUN-006 弹性收敛（`run_006/output/round_5_catalyst.md`）。

### 决策 7：不预设第四种关系类型

替代方案：在三种基础关系类型（互补/同向/对冲）之外预设更多类型。

实验依据：RUN-006 催化自发识别了"同域定位"——P03↔P04 不构成互补、同向或对冲，但两者的组合为第三方（需求方）提供了超出各自之和的信息价值。这个关系类型在 R2-R3 自然涌现，不需要预设。三种基础类型足够覆盖催化的识别需求，更复杂的关系模式由催化在运行时从数据中归纳。预设关系类型会限制催化的发现能力。

证据来源：RUN-006 开放探针 #1（`run_006/output/audit_scaling_and_open.md`）。

## 工程化规格

### 输入输出接口

```
Formulation 接口：
  输入：原始需求文本（自然语言）+ 需求方 Profile（文本/文件）
  输出：T/I/B/E 编码（结构化 markdown，≤300 字）+ 数据审计表
  评级：A（四参数均充分）/ B（T+I 有，B/E 缺）/ C（仅身份标签）
  C 级 Profile 不建议进入结晶

Endpoint 接口：
  输入：T/I/B/E 编码 + endpoint prompt + Profile 全文或相关片段
        + 催化反馈（R2+）
  输出：三维投影（能力/方向/边界），markdown 格式
  R1：完整投影；R2+：增量或"我没有新的信息了"
  预期大小：2-4KB/轮

Catalyst 接口：
  输入：N 个端侧投影 + 上轮催化输出 + T/I/B/E 编码
  输出：全对覆盖的关系分析 + 翻译 + 缺口 + 被忽略信息
        + 收敛判断（R3+）：CONTINUE 或 CONVERGED
  预期大小：3-5KB/轮（3 人）；5-8KB/轮（5 人）

Plan 接口：
  输入：全部结晶轮次数据 + 参与者 Profile + T/I/B/E 编码
  输出：协作全景 + 各参与者方案 + 协作顺序 + 残余张力
        每条声称标注来源（Round N + Agent）
  预期大小：5-15KB
```

### 部署架构

**编排层（状态机）**：管理结晶过程的轮次、路由、收敛判断。每个结晶过程 = 一个状态机实例。状态机预设 max_rounds = 7，参与者上限 8 人（28 对，催化处理能力的实测安全边界内）。并发上限由 API rate limit 决定，非计算资源。

**催化层（服务端 API 调用）**：使用最强模型（opus/sonnet）。催化的智能程度直接决定信息差识别和跨语义翻译的质量——这是系统中最值得投入算力的位置。每轮一次调用，token 上限根据参与者数量动态调整。

**端侧层（三种接入模式）**：

模式 A——用户自带 Agent。用户通过 MCP 接口接入自己的 Agent（SecondMe 级别的深度个性化模型，或任何能遵循投影格式的 LLM 实例）。协议不关心端侧实现——只关心投影输出格式。

模式 B——服务端代理。平台为用户的 Profile 构建端侧代理。大 Profile（>50K）预处理为分块 → embedding → 向量索引，每个新张力触发 tension-aware 检索 → 组装 10-15K 端侧上下文。700K Profile（如 P03）的信息压缩比 > 180:1（RUN-005/006 实测）。端侧 API 调用使用中等模型（sonnet）。

模式 C——混合模式。同一结晶过程中部分参与者使用模式 A，部分使用模式 B。协议层不区分——催化只看到投影输出，不知道也不需要知道端侧实现细节。

**Profile 预处理**：小 Profile（<50K）可直接全文注入 endpoint prompt。大 Profile（>50K）必须预处理：分块 → embedding → 向量索引 → tension-aware 检索 → 组装上下文。检索策略将 T/I/B/E 编码作为检索查询的一部分，确保检索结果与当前张力相关。

### 成本模型

```
单个 3 人结晶过程（5 轮）：
  Formulation：1 次 API 调用 ≈ $0.05-0.10
  催化：5 轮 × 1 次 ≈ $0.50-1.00
  端侧（模式 B）：3 人 × 5 轮 = 15 次 ≈ $0.45-0.75
  Plan 生成：1 次 ≈ $0.10-0.20
  总计：$1.10-2.05/process

单个 5 人结晶过程（5 轮）：
  催化：5 轮 × 1 次 ≈ $0.75-1.50（催化输入更大）
  端侧（模式 B）：5 人 × 5 轮 = 25 次 ≈ $0.75-1.50
  总计：$1.65-3.30/process

30 个并发 process：$50-100/批次
```

成本的主要驱动因素是催化层——催化消费全部端侧投影，token 量随参与者数量线性增长。端侧成本随参与者数量线性增长但单次调用 token 量较小。Profile 预处理（embedding）成本可忽略。

## 从推导链中沉淀的认知（更新版）

v1 文档记录了概念膨胀和修剪过程中沉淀的认知。6 次实验后，部分认知被修正，部分被加固。

### 被修正的认知

"Formulation 不是预处理，是循环中自然发生的"——RUN-002 证明这是错的。Formulation 质量是根部变量，必须在循环前完成且可审计。Formulation 从被砍掉的概念中回到了协议的核心位置。

"Plan 不再被生成，自然涌现"——实验证明 Plan 需要独立生成器。来源标注、依赖链、残余张力的操作性都需要显式处理。

"协议不假设端侧有任何特定能力"——实验测量到端侧投影质量直接决定催化的注意力分配和整体结晶质量。协议对端侧有明确的能力期望（信息函数不变量、T/I/B/E 对准、三维投影格式），并提供参考实现。

"端侧用自然语言回复，无格式要求"——endpoint_v2 要求三维结构 + T/I/B/E 对准 + 信息函数不变量。格式约束不杀信息——杀噪声。

### 被加固的认知

"协议止于可见性"——6 次 RUN 中催化始终不推荐方案。方案生成是循环结束后的独立步骤。

"残余张力递归"——RUN-005/006 的残余张力足够具体到可触发新一轮场发现。残余与原始张力在协议层面完全同构。

"催化消除信息差"——信息差的两个来源（传输问题 + 翻译问题）在实验中都被观测到。催化在信息丰富的轮次（RUN-005 R4、RUN-006 R3）展示了全程最深的翻译能力。

"三种关系类型逻辑完备"——6 次 RUN、数十对配对中，催化识别的所有关系均可归类为互补、同向或对冲。RUN-006 的"同域定位"看似第四种类型，但它描述的是组合对第三方的信息价值，不是两个参与者之间的直接关系——不构成对三种基础类型完备性的挑战。

### 新增的认知

催化不是被动观察者——它的框架构建是主动的认知操作，可以重新激活已收敛参与者的数据。

端侧投影函数对信息环境不敏感——f(Profile, Tension) 不是 f(Profile, Tension, OtherParticipants)。这是可工程化依赖的不变量，意味着端侧可以并行执行且结果可复现。

收敛速度由边界意识决定，不由 Profile 体量决定——边界诚实的 Agent 收敛更快。

## 迭代轨迹

```
RUN-001  form_v0 + cat_v1 + end_v0
         6 轮未收敛。endpoint_v0"参加讨论"激活对话默认行为，70% 噪音。
         诊断 → 三重投影模型推导（DESIGN_LOG_007）

RUN-002  form_v0 + cat_v2 + end_v1
         3 轮未收敛。catalyst_v2 有效，endpoint_v1 消除 cosplay。
         但 formulation_v0 800 字心理画像污染全部下游。
         诊断 → formulation 是根部变量，必须改

RUN-003  form_v1 + cat_v2 + end_v1

         但 endpoint_v1 膨胀 70-80%，催化注意力偏斜（P03↔P04 零分析）。
         诊断 → 三项补丁（信息函数 / 匿名化 / 全对覆盖）

RUN-004  form_v1.1 + cat_v2.1 + end_v2（v2 基线）
         5 轮收敛。三项补丁全部生效。endpoint_v2 压缩 53%。
         催化 R1 即发现 P04↔P03 方法论对冲（RUN-003 遗漏）。
         决策 → v2 基线固化，停止 prompt 迭代

RUN-005  v2 基线 + D02 新需求 + 差异化参与者
         5 轮收敛。协议普适性验证通过。

RUN-006  v2 基线 + D02 + 5 人场（10 对）

```

每次迭代的因果链（问题 → 修改 → 效果）构成了设计决策的实验依据。RUN-001 的 cosplay、RUN-002 的根部污染不是黑历史——它们是当前设计的因果锚点。
