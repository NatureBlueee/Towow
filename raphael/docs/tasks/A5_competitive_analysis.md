# A5 — 竞品/参考系统对标分析

## 为什么做这件事

通爻提出了"响应范式"来替代"搜索范式"。外界会问："这和 AutoGen 有什么区别？和推荐算法有什么区别？"——如果我们自己说不清楚，别人更说不清楚。

这个分析不仅是为了"证明独特"，更是为了**精准识别独特价值在哪**。可能的发现包括："别人已经在做 X 了，但没人做 Y"——这就是通爻的差异化定位。

## 你要回答什么问题

**核心问题**：通爻网络和以下系统/框架的**范式级别**差异是什么？

**需要对标的系统**（三类）：
1. **多 Agent 框架**：AutoGen、CrewAI、MetaGPT、LangGraph、A2A Protocol
2. **匹配/推荐系统**：协同过滤、LinkedIn、Upwork、Boss 直聘
3. **去中心化/新范式**：Fetch.ai、SingularityNET、SecondMe

**子问题**：
- 每个系统的核心假设是什么？（它预设了什么？）
- 哪些能处理"未知需求"？（搜索范式 vs 响应范式的分水岭）
- 通爻的"投影"和其他系统的"画像/Profile"有何本质不同？（投影是函数调用，画像是静态存储）
- 是否有系统已经在做通爻想做的事？（如果有，通爻的增量在哪？）
- 各系统的 context 来源是什么？（单一 LLM vs 分布式 Agent 各自的 Profile）

### 概念翻译提示
- "投影" ≠ "画像"：投影是函数调用（每次调用可能不同），画像是静态存储
- "共振" ≠ "匹配"：共振是端侧自主判断，匹配是中心化计算
- "协商" ≠ "排序"：协商产生新的整体，排序只是从已有中选

## 子任务分解

### A5.1 — 收集对标系统信息
**描述**：为三类对标系统收集核心信息——官方文档、白皮书、架构图、核心卖点。
**依赖**：无
**交付**：每个系统的 1 页信息摘要

### A5.2 — 分析各系统的核心假设
**描述**：提取每个系统的隐含假设。比如：AutoGen 假设"任务可以分解为步骤"；LinkedIn 假设"用户知道自己要找什么"。
**依赖**：A5.1
**交付**：假设对比表

### A5.3 — 制作多维度对比矩阵
**描述**：从范式层面（不是功能层面）对比。维度包括：context 来源、是否能处理未知需求、计算架构（中心化/分布式/端侧）、核心操作（搜索/投影/匹配/协商）。
**依赖**：A5.2
**交付**：对比矩阵表格

### A5.4 — 撰写差异分析
**描述**：基于对比矩阵，深入分析通爻的独特价值。要公平——承认别人做得好的，指出通爻的真正差异化点。
**依赖**：A5.3
**交付**：差异分析文章（5000-8000 字）

### A5.5 — 提炼定位一页纸
**描述**："通爻是什么，不是什么"——500 字以内，任何人 1 分钟读完就知道通爻和别人有什么不同。
**依赖**：A5.4
**交付**：定位一页纸（500 字）

## 做完了是什么样

1. **对标分析报告**（5000-8000 字）：含对比表和差异分析
2. **一页纸定位**（500 字）："通爻是什么，不是什么"

**质量标准**：
- 至少覆盖 8 个对标系统（三类各至少 2 个）
- 范式级对比，不是功能列表
- 有引用和出处
- 公平对待各系统（不贬低别人来抬高自己）

## 你必须遵守的

- 公平对待各系统——发现别人做得更好的地方也是有价值的
