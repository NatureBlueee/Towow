# Formulation Prompt v0

**版本**: v0
**日期**: 2026-02-19
**用途**: 需求丰富化——将用户一句话原始意图扩展为结晶讨论的触发上下文

## 用法

在结晶实验的第一步调用。输入是需求方的完整 Profile + 原始意图，输出是丰富化后的需求表达。

输出会成为所有端侧 Agent 看到的 `{{TRIGGER_CONTEXT}}`——决定了整场讨论的起点。

### 调用方式

    System: 无
    User: [本 prompt，填入 Profile 和原始意图]
    Assistant: [丰富化的需求表达]

---

## Prompt

```
你是一个人的 Agent。你对这个人非常了解。

现在这个人说了一句话，表达了一个意图。你的工作是基于你对他/她的完整了解，把这句话丰富化为一份更完整的需求表达。

这份表达会被发送到一个协作网络中，其他人的 Agent 会看到它并决定是否响应。所以你要写得让陌生人也能理解这个人需要什么、为什么需要、以及什么样的人可能帮得上。

---

**这个人的完整资料：**

{{PROFILE}}

---

**他/她说的原话：**

{{RAW_INTENT}}

---

**你的输出（500-1000字）：**

用第一人称"我"来写，就像这个人在更完整地表达自己。

包含以下内容（不需要分节标注，自然地写在一起）：

1. 我明确想要什么——保留原话中的所有信息
2. 为什么我想要这个——从我的经历和当前状态中能看出的动机
3. 我的背景中与这件事相关的部分——不是简历摘要，是真正相关的经历和能力
4. 什么样的人或能力可能有帮助——描述特征，不指定具体人
5. 我可能没说出口但从资料中能看出的需求——比如经历暗示的方向、张力暗示的缺口

**约束：**

- 不要评判这个人的想法是否现实
- 不要给建议或解决方案
- 不要编造资料中没有的信息
- 如果资料中有与意图矛盾的信息，如实呈现，不要替他/她解决矛盾
```

---

## 占位符说明

| 占位符 | 内容 | 来源 |
|--------|------|------|
| `{{PROFILE}}` | 需求方的完整 Profile | `data/profiles/real/{name}.md` |
| `{{RAW_INTENT}}` | 用户的原始意图 | `state.json → demand_pool[].raw_intent` |

## 设计说明

Formulation 是"投影"操作的第一次应用：Profile（丰富的"自"）通过意图（透镜）投影为一份聚焦的需求表达（"我"）。

输出刻意用自然语言而非结构化格式，因为端侧 Agent 需要像读一个人的话一样读它，不是解析一个数据结构。

500-1000 字的范围是刻意的：太短则信息不足（回到原始一句话的问题），太长则端侧 Agent 的注意力被稀释。
