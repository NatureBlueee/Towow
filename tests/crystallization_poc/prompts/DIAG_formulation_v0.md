# Formulation v0 问题诊断

**日期**: 2026-02-19
**来源**: RUN-002 逐轮审查（用户 + arch skill）
**严重度**: Critical — 链条根部污染

---

## 现象

用户作为外人读完 formulation 输出后的原话："我并不觉得很好，在看完这些后作为外人我并没有更理解这个人的需求，就是看了一篇emmm的东西，没有看到，真的"

## 输入

李默妹的原始意图（2行，语法不完整）：
> 我是聋人，艺名是李默妹，Entj射手座是一个没有基础「走向星球、长寿科技、人工智能安全性能管理体系及个人提升」的普通人，但是我想要普通人通过 AI Agent系统的学习四个方面领域整结创作新的有用技术发展未来科技，慢慢来研究和发现…..

## 输出（formulation_v0 产出的问题）

800字心理画像，包含：
- 李默妹的宗教脱离经历（原始意图未提及，从Profile推断）
- "被认可的需要"的心理分析（明确标注为"没有直接说出口的东西"）
- ENTJ人格描述
- 聋人身份叙事
- 四领域"咬合"的感性描述

## 未回答的核心问题

1. **她到底想要别人帮她做什么？** — 未解析
2. **"整结创作新技术"是什么意思？** — 写博客？做产品？搭系统？发论文？未追问
3. **"四领域咬合"在她脑子里是什么？** — 她自己清楚吗？Formulation没有追问
4. **她要找的是什么角色？** — 导师？同路人？工具提供者？验证者？未区分

## 根因分析

### 问题1：边界不清

Formulation 的工作应该是"把模糊意图变成清晰的张力"。但 prompt 没有约束它只做这件事。结果它做了更容易的事——基于 Profile 写一份关于这个人的完整叙事。

### 问题2：绕开了难题

原始意图**真的很模糊**。"整结创作新的有用技术"这个表述几乎无法直接解析。好的 formulation 应该**承认这个模糊性**并尝试解构，而不是用 800 字的流畅叙事掩盖模糊。

### 问题3：过度推断

从 Profile 中提取的信息（宗教经历、被认可需要）被当作需求的核心维度写进了张力描述。这些推断可能准确也可能不准确，但它们被呈现为事实，直接主导了下游注意力分配。

## 下游影响

1. **端侧 Agent** 拿到 800 字心理画像作为 tension_context → 投影也在回应画像（"我尊重你不被定义"）而非具体需求
2. **催化** 把"宗教经历未被回应"列为"被忽略的信息"，把"三方都没回应被认可需要"列为"隐藏的对冲" → 注意力被分配给 formulation 的推断而非需求本身
3. **最终方案** 建立在被偏移的关系分析之上

## 下一步

读 `formulation_v0.md` prompt → 理解为什么它倾向写叙事 → 设计 formulation_v1：
- 核心任务：解析模糊意图，不是扩写
- 输出约束：短（<200字？），聚焦"要什么+约束条件"
- 对模糊性诚实：如果意图无法解析，明确标注哪些部分是模糊的
- 不做心理分析：Profile 事实可以引用，但不替需求方推断动机
