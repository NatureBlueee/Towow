## 一、五定向探针结果

**探针1 B分离度**: 全程完全分离，零衰减。10对配对中催化始终将每条关系对准具体B子问题，五轮无任何B1/B2/B3混合处理。比RUN-005证据更强（10对 vs 3对）。

**探针2 P03漂移**: 全程零漂移。P03在有同域参与者P04（心灵奇记，同为情感类AI产品）的环境下，投影始终锁定在D02的C端市场/产品维度，未激活任何D01教育工具维度内容。信息环境差异（P04/P08的加入）只在张力框架内激活了P03 Profile的不同子集（如OneThing onboarding经验），没有超出D02范围。

**探针3 收敛因**: 数据自然耗尽为主导，边界精度饱和为辅。与RUN-005不同（RUN-005以边界精度饱和为主导）。R3催化构建的B3四种断层框架一次性激活了所有参与者的最后一批相关内容，R4时4/5同步收敛。P01出现"R3收敛->R4重新激活"的新行为——催化框架更新重新定向了已有数据。

**探针4 T1修正**: 催化展示了三种修正行为：(1) 视角提升（P08 R1被重新定位为目标客户样本），(2) 约束类型区分（P04 R2区分能力缺失和时间约束），(3) 维度重新定向（P01 R3->R4已有数据对准新维度）。第三种是5人场新行为。

**探针5 催化带宽**: 峰值在R3（非RUN-005的R4）。R3构建了B3四种断层路径框架+翻译7三层合并图，是整个RUN-006信息密度最高的一轮。R4结构性下降（零翻译），R5进入收敛态。催化选择性注意自然发生：10对中4对低信号关系被快速判定"无关系"，不投入翻译资源。

## 二、两个因果归因

### P01 R4 重新激活

**结论**: 框架激活假设成立。P01明确引用"Round 3催化建立了B3的四种断层路径"作为重新激活的触发器。R1已投出的机构弃用数据被重新定向到B3维度（原在B1）。这是催化框架构建的"语义槽位匹配"效应——新框架创造了新的对准位置，匹配已收敛参与者的已有数据。不支持"信息传递"假设（P01未引用其他参与者内容），不支持"延迟投影"假设（是旧数据重新定向，非新内容）。

### R4 同步收敛

**结论**: 数据自然耗尽假设成立，但R3催化框架间接驱动了同步性。4个收敛声明的核心都是"Profile中与张力相关的内容已投完"或"剩余内容无关"。P05和P08有微弱的框架引用，但都是将催化框架用作确认工具而非收敛驱动因素。深层因果链：R3催化框架一次性激活了所有人的最后一批数据 -> R4全部投完 -> 同步收敛。

**演化轨迹**: R1二值化(5:5) -> R2三层化(深度/中度/零声明并存) -> R3集中到中度+浅度 -> R4全面压缩。分布形状从"尖峰+零谷"演化为"全部低平"。

**关键发现**: R1的初始二值化判断精确对应最终关系图谱。催化在R1就几乎没有遗漏高价值关系。注意力预算分配策略自然形成为"前期发现(R1-R2)，后期深化+关闭(R3-R4)"。

## 四、开放探针：框架外观测值

1. **催化发明新关系类型"同域定位"**: P03-P04被标注为"同域定位，不构成需催化指出的信息差"——既非互补、同向、对冲。本质是组合信息价值>各自之和，但消费者是第三方(P02)。
2. **催化产出"四种断层路径"分类框架**: B3从单一维度展开为建造者型/网络者型/分散稀释型/采购未激活型四条路径。催化自发形成的二阶概念，超越本次实验的通用价值。
3. **弹性收敛判断**: R5催化主动偏离"连续两轮后者"的机械规则，基于5/5全员无新内容的实质判断提前发出[CONVERGED]信号。
4. **P01-P05"三段配置"关系结构**: 同一参与者(P01)在同一对关系中扮演双重角色(数据提供方+整合输出方)，预设互补模型无法表达。
5. **端侧"元定位修正"**: P03在R3拒绝催化赋予的"目标用户样本"定位，提出替代定位"没有结构性增长支持的代价样本"。参与者对催化概念框架的反向干预。

## 八、对 Module 2 的设计参数提取

以下设计参数从三份审计报告中提取并去重：

### 端侧 (endpoint)

- **endpoint_v2 的信息函数不变量在 10 对场景下完全稳定**，不需要为 scaling 做 B 分离相关的修改
- **投影张力锁定在 5 人场+同域参与者条件下完全稳定**，不需要为"同域参与者相互拉偏"添加额外防护
- **端侧投影函数对信息环境不敏感是可工程化依赖的不变量**，R1 投影可并行执行无需传递参与者列表
- P01 的变异提示 endpoint prompt 有一定随机性宽度（同一能力域的不同切面），若需可复现性可增加确定性约束

### 催化 (catalyst)

- **选择性注意是自然行为，不需要显式编程**。"逐对检查 + 允许'无关系'输出"格式天然支持 scaling
- **注意力预算的自然衰减曲线是设计参数**: R1-R2 发现窗口 -> R3-R4 深化/关闭窗口。10人/45对场需要 R1-R2 就实施分层处理策略
- **催化框架构建具有"重新激活"效应**：收敛判断不能基于单个参与者声明收敛的简单计数，需等催化框架稳定后再确认
- **翻译峰值从 R4 前移到 R3**（5人场信息负荷增大），有效翻译窗口可能更短但密度更高
- **催化效率可能随参与者数量增加而非下降**：更多参与者 = 更丰富框架 = 更快数据激活 = 更快收敛（待更多 RUN 验证）

### 收敛判断

- **不能在单个参与者声明收敛时关闭其输入通道**——催化框架更新可能重新激活已收敛参与者
- **全员收敛状态检测应作为快速终止条件**，当M/M参与者全部声明无新内容时无需等待第二轮确认
- **5 轮是较为稳定的收敛窗口**，可作为默认最大轮次（5-7轮），除非 Profile 体量极大

### Plan 生成器

- **E 层约束应成为标准产出维度**，将催化"被忽略的信息"作为残余张力候选池输入源
- **角色分层是 5+ 人场的必需结构**（数据提供者/整合者/目标用户原型三层）
- **P01 整合节点瓶颈是 scaling 风险**，需设计"整合节点自动分裂"机制
- **"被忽略信息"升级为残余张力的判断标准**："是否构成协作启动或持续的前置条件"，应显式编码
- **Plan 应自底向上生成方案结构**（从催化关系图谱和张力结构），不预设模板

### 关系模型扩展

- 新增关系类型"同域定位"（组合信息价值>各自之和，消费者是第三方）
- 关系图谱需支持同一节点在同一边上的多角色（如 P01 的数据提供+整合双重角色）
- 催化产出的"二阶概念"（分类/命名/结构化框架）需被 Plan 识别和保留
