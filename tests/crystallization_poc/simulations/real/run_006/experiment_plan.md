## 定位

RUN-006 不是 v2 基线的及格考试。是 Module 2 催化架构的 X 光扫描。

从 5 人场（10 对）中提取四个 Module 2 工程决策的设计参数：
1. 催化的注意力分配函数
2. 低信号对的处理架构
3. 端侧在信息环境膨胀下的稳定性
4. Plan 生成器的 scaling 行为

三种结局（通过/部分通过/崩溃）都是有价值的数据点。价值不在判定结果，在从行为模式中提取设计参数。

## 四个观测维度

### 维度一：催化隐式优先级排序

催化面对 10 对时的自然注意力分配 = Module 2 注意力分配算法的原型。

**测量方法**: 每轮催化输出中，10 对的处理深度（翻译段字数 + 语义层级分类: 命名/条件性/结构性）。排序后观察分布形状：
- **均匀分布** → 穷举仍可行，v2 基线直接 scale
- **长尾分布** → 催化在自然做注意力分配，提取排序规则（按关系强度？信息新颖度？B 对准度？）
- **二值化分布** → 催化在做硬剪枝，高信号对精处理 + 低信号对跳过

**Module 2 映射**: 催化自然排序与"对 plan 贡献大"的事后排序的相关度 → 决定 Module 2 是否需要从头设计注意力分配，还是只需将催化的隐式排序显式化。

### 维度二：低信号对处理模式

P04 相关的 4 对（P04×P03/P01/P05/P08）大概率是低信号对。催化的自然处理方式直接决定 Module 2 架构选择：

- **模式 A（显式零关系）**: 催化输出"P04×P05 未发现直接关系" → Module 2 保持逐对穷举 + 零关系声明
- **模式 B（跳过）**: 催化跳过低信号对 → Module 2 需要前置信号预筛步骤
- **模式 C（聚类声明）**: 催化输出"P04 与本场其他参与者直接关系均较弱" → Module 2 输出架构从"逐对"变"逐人 + 高信号对精检"

**关键约束**: 不在 prompt 中预设催化应该怎么处理低信号对。让它自然选择，观测选择。

### 维度三：端侧信息环境效应

3 人场催化反馈涉及 2 个其他参与者。5 人场涉及 4 个。端侧收到的催化信息量翻倍。

**测量方法**: P03/P01/P05 三个老参与者在 RUN-006 vs RUN-005 同轮次的投影差异度。

- **差异 ≈ 0** → 端侧对信息环境不敏感。endpoint_v2 的张力锁定机制足够强
- **差异 > 0，方向一致**（仍对准 D02 张力）→ 新参与者提供了有价值的新刺激
- **差异 > 0，偏离张力**（开始投射对 P04/P08 的能力而非对准 D02）→ 信息环境污染，endpoint 需要"抗噪"机制

这是探针 2（张力锁定）的 scaling 版本——不是催化引导漂移，是信息环境复杂度增加导致的漂移。

### 维度四：Plan 生成器 scaling

3 人 plan: 线性依赖链。5 人 plan: 潜在网状依赖（复杂度 O(N²)）。

**测量方法**: Plan 的结构复杂度 + 可执行性。
- 依赖图是否仍有清晰的行动路径？还是退化为关系罗列？
- 5 个参与者方案是否每个都有独立的角色/贡献/成本/依赖？
- 残余张力是否因参与者增加而减少？

## 审计框架

### 保留探针（适配 5 人版）

| 探针 | RUN-005 版本 | RUN-006 适配 |
|------|-------------|-------------|
| 1. B 分离度 | 3 人是否将 B1/B2/B3 混为一体 | 5 人是否将 B1/B2/B3 混为一体（新参与者是否影响老参与者的 B 分辨率） |
| 2. P03 漂移 | P03 是否漂移回 D01 元素 | P03 是否漂移回 D01 元素 **且** 是否被新参与者拉偏 |
| 3. 收敛因 | 3 人收敛因序列 | 5 人收敛因序列（P04/P08 的收敛行为是新观测） |
| 4. T1 修正 | 催化修正 T1 偏移轨迹 | 同上 + T1 是否因更多参与者而被更好覆盖 |
| 5. 催化带宽 | 翻译段语义深度轨迹 | **每对** 翻译段语义深度轨迹（核心 scaling 指标） |

### 新增探针

| 探针 | 测量目标 | 方法 |
|------|---------|------|
| 6. 开放探针 | 预设答案空间外的观测值 | "本轮数据中是否有任何观测值不在上述探针的预设答案空间内？" |
| 7. Scaling-A: 催化注意力分布 | 10 对处理深度的分布形状 | 每轮 10 对翻译段字数排序 + 分布分类（均匀/长尾/二值化） |
| 8. Scaling-B: 端侧信息环境 | 老参与者投影 vs RUN-005 差异度 | P03/P01/P05 同轮投影对比 |
| 9. Plan 产物审计 | 关系传递完整性 + 依赖链可执行性 | 催化最终输出 vs plan 对比（主代理直接执行） |

### 审计执行方案

- R1-R5（或更多轮）: 每轮派 1 个 subagent 读该轮全部输出，回答探针 1-8
- Plan 审计: 主代理直接执行（对比催化收敛输出 vs plan.md）
- 综合: 主代理汇总所有探针轨迹 + 四个观测维度的参数提取

## 预期结果与 Module 2 决策映射

| 观测结果 | Module 2 决策 |
|---------|--------------|
| 催化注意力均匀分布 | 逐对穷举保持为默认架构，无需注意力分配机制 |
| 催化注意力长尾分布 | 提取排序规则，显式化为 Module 2 的注意力分配函数 |
| 催化硬剪枝（跳过低信号对） | Module 2 需要前置信号预筛步骤 |
| 低信号对模式 A（显式零关系） | Module 2 逐对输出，包含零关系声明 |
| 低信号对模式 C（聚类声明） | Module 2 输出架构重设计：逐人 + 高信号对精检 |
| 端侧投影无漂移 | endpoint_v2 直接 scale |
| 端侧投影偏离张力 | endpoint 需要抗噪机制（信息环境过滤） |
| Plan 保持行动路径 | plan_v0 直接 scale |
| Plan 退化为关系罗列 | plan 需要独立的路径规划步骤 |

## 风险与缓解

| 风险 | 概率 | 缓解 |
|------|------|------|
| max_tokens 截断催化输出 | 中 | Agent Teams 模式下 sonnet 默认足够大。首轮后检查催化输出完整性 |
| 收敛轮次拉长（>7） | 低 | 如超 7 轮仍未收敛，手动终止并分析原因（这本身是有价值数据） |
| P04 投影极稀疏（几乎无内容） | 中 | 预期行为。稀疏投影本身是"低相关参与者"的测试数据 |
| 催化注意力完全偏向老 3 对 | 低 | 如发生，说明催化有"熟悉度偏差"，Module 2 需要消偏机制 |
