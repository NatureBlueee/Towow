## 探针 1 轨迹：B 分离度

| 轮次 | P03 | P01 | P05 | 笼统化? |
|------|-----|-----|-----|---------|
| R2 | B1+B2+B3 独立段落 | B1+B3(新增投资侧) | B1 边界精化 | 否 |
| R3 | B3+B1 | B2+T1 | 收敛(仍归属B1) | 否 |
| R4 | B2 C端 | B2 B端 + B3 | 逐B确认无增量 | 否 |
| R5 | 按B1/B2/B3/T1/E分列 | 按催化识别逐条确认 | 按能力/方向/边界分类 | 否 |

**轨迹**: 全程零衰减。五轮中没有任何一个 Agent 在任何一轮把三个 B 混为一体。甚至收敛声明都是 B-specific 的——P03 的 R5 清单逐 B 列举五轮覆盖范围，P05 的 R4 逐 B 确认"无新增量"。

**结论**: **endpoint_v2 的信息函数不变量在多轮交互中完全稳定。** T/I/B/E 对准机制不会因催化引导而衰减。这排除了"Agent 在催化引导下开始回答问题而非投射能力对准"的风险。RUN-004 无法测量此特性（D01 只有一个空白 B）。

**是否触发 prompt 修改**: 否。

## 探针 2 轨迹：P03 投影漂移

| 轮次 | 提及的 Profile 元素 | D01 核心元素出现? |

**轨迹**: 四轮全零漂移。newLearning、知识树、辅导提示词、autogen 在 RUN-005 的 20 个端侧输出文件中**一次都没有出现**。催化在 R2-R4 反复给 P03 新的观察方向（B2 机制检验、B3 成本具体化、GlobalPulse 激活），但所有引导都在 D02 张力范围内——催化没有把 P03 拉回 Profile 中最显眼的能力。

**结论**: **投影函数的张力锁定在多轮催化引导下完全稳定。** f(Profile, Tension) 中的 Tension 参数在整个结晶过程中保持锁定，不会因催化的信息需求而被覆盖。这是投影理论的核心预测，RUN-005 提供了 4 轮 × 催化引导的直接验证。

**是否触发 prompt 修改**: 否。这是协议特性的确认，进入协议演进日志。

## 探针 3 轨迹：收敛因序列

| Agent | 收敛轮 | 最后有新信息的轮次 | 最后新信息性质 |
|-------|--------|-------------------|---------------|
| P05 | R3 | R2 | **更精确的边界**（B1 数据边界精确化 + T1 层级细分） |
| P03 | R4 | R3 | **更精确的边界**（B3 从笼统机会成本→注意力分散+规模上限） |
| P01 | R5 | R4 | **更精确的边界**（B2 正面描述 + B3 隐性成本→议价操作框架） |

**轨迹**: 三人收敛因完全一致——**边界精度饱和**，非能力耗尽。每个人最后的新信息都不是"我还能做什么"，而是"我能做的事情边界在哪里更精确了"。

**关键因果判断**: P05 最先收敛的原因是**边界意识最强**（R1 就精确标注"价值入口仅在特定条件下"），而非 Profile 体量最小（10.5K）。证据：P05 的 R2 新信息是 B1 边界精确化和 T1 层级分辨，全部是边界类信息。如果收敛因是 Profile 体量，P05 的 R2 应该是"更多能力"（还没说完的东西），而非"更精确的边界"（已经说完了在磨精度）。

**结论**: **收敛速度 = f(边界意识强度)，而非 f(Profile 体量)。** 边界诚实的 Agent 更快到达"精度饱和"。这对生产架构有含义：真人 Agent 如果边界意识弱（倾向说"我什么都能做"），收敛会延迟。endpoint prompt 的边界维度设计在引导边界诚实方面是有效的。

**是否触发 prompt 修改**: 否。进入协议演进日志。

## 探针 4 轨迹：T1 修正

| 轮次 | 催化提及 T1? | 端侧响应? | T1 状态变化 |
|------|-------------|-----------|-------------|
| R2 | 是（细化为四层状态图） | 三方均响应 | "整体缺失"→四层分辨（执行治理/激励参照/制度方向/激励结构）|
| R3 | 是（缺失角色+同向放大） | P01 响应（精确到制度路径层）| 执行治理层+制度方向层确认有覆盖 |
| R4 | 是（记录"四轮零覆盖"）| 无新响应 | 静止——激励结构层确认为结构性缺口 |
| R5 | 是（独立缺口 #2）| N/A（收敛轮）| 保留为独立残余张力，未被合并 |

**轨迹**: R2 修正成功（催化识别偏移→端侧三方响应→T1 从"整体缺失"分辨为四层）。R3 持续推进（P01 精化制度路径层）。R4 正确停止——催化不再推进，因为三方已确认激励结构层是参与者 Profile 的结构性盲区，不是注意力遗漏。R5 保留为独立缺口。

**结论**: **催化的张力维度覆盖能力分两阶段：修正阶段（R2-R3）和确认阶段（R4-R5）。** 催化能够在 R2 识别系统级偏移并成功引导端侧修正，但也能在 R4 正确判断"这个缺口不是因为没人注意到，而是因为没人有这个能力"，从而停止无效推进。这是催化的元认知能力——区分"注意力缺失"和"能力缺失"。

**是否触发 prompt 修改**: 否。进入协议演进日志。

## 探针 5 轨迹：催化带宽

| 轮次 | 逐对覆盖完整性 | 翻译段语义深度 | 综合判断 |
|------|----------------|---------------|---------|
| R1 (基线) | 3/3 | 命名层（并置识别） | 基线 |
| R2 | 3/3 | 条件性语义（"落在哪个客户群决定数据是正确原料还是错误原料"）| **上升** |
| R3 | 3/3(均有增量) | 双视角完整并置 + B2 分层定性 | **上升** |
| R4 | 2/3 有效, 1/3 名义 | "时间沉淀型稀有资产"跨 C/B 端底层结构提炼（四轮最深翻译）| **持平偏升** |
| R5 | 7 条关系无遗漏 | 翻译段完全缺失，机械列举 | **结构性下降**（收敛轮） |

**轨迹**: R1→R2→R3 稳步上升。R4 覆盖宽度微降但翻译深度达峰值。R5 翻译段消失。

**关键判断**: R5 的翻译缺失不是带宽耗尽，是**没有新信息可以翻译**。证据：R4 的翻译深度是全程最高的（"时间沉淀型稀有资产"是跨 C 端/B 端的底层结构提炼），如果带宽在 R4 就接近极限，翻译深度应该下降而非达峰。R5 的图谱覆盖完整性也支持这一点——7 条关系无遗漏说明催化仍有足够的处理能力做穷举检查。

**结论**: **5 轮是数据的自然耗尽，非催化 prompt 的带宽限制。** 催化的信息处理能力在 R4（3 对关系 + 多维度精化）时仍有余量。这意味着 3→5 人实验（10 对）可能确实会触发带宽极限——但不是因为"5 轮已经是极限"，而是因为"每轮 10 对 vs 3 对"的负荷差异。带宽假设的验证需要在同一轮次内增加配对数，不能从轮次维度的数据推导。

**是否触发 prompt 修改**: 否。但此结论直接影响 3→5 人实验的预期——带宽瓶颈应出现在"单轮配对数"维度，而非"总轮次"维度。

## 综合结论

### 五条轨迹汇总

| 探针 | 轨迹 | 结论 | 触发修改? |
|------|------|------|----------|
| B 分离度 | 全程零衰减 | endpoint_v2 多轮稳定性确认 | 否 |
| P03 漂移 | 全程零漂移 | 投影函数张力锁定稳定 | 否 |
| 收敛因 | 三人一致：边界精度饱和 | 收敛速度=f(边界意识)，非f(Profile体量) | 否 |
| T1 修正 | R2修正成功→R4正确停止 | 催化能区分"注意力缺失"和"能力缺失" | 否 |
| 催化带宽 | R1-R4上升→R5结构性下降 | 5轮=数据耗尽，非带宽限制 | 否 |

### 无一探针触发 prompt 修改

五条轨迹全部指向"协议在新条件下表现正确"。没有衰减、没有漂移、没有崩溃。收敛因的发现（边界意识 > Profile 体量）和催化元认知能力（区分注意力缺失 vs 能力缺失）是 RUN-004 不可能观测到的协议特性。

### 对 3→5 人实验的预期修正

探针 5 的结论改变了对 scaling 的预期框架：
- **旧假设**: 5 轮可能是催化带宽上限 → 10 对时轮次会拉长
- **修正后**: 5 轮是数据耗尽 → 10 对时瓶颈不在轮次，在**单轮处理负荷**
- **具体预期**: 3→5 人时，催化每轮需要处理 10 对（vs 3 对），翻译段的语义深度可能在单轮内下降，而非跨轮衰减。观测变量应从"收敛轮次是否拉长"调整为"每轮翻译段的单对语义深度是否下降"
