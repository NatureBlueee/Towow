## 一、投影差异度验证——P03 跨张力对比

这是 RUN-005 最重要的实验观测点。

**RUN-004 中的 P03（面对 D01："帮新手学习"）**

能力投影核心：newLearning 知识树提示词、一对一辅导提示词、autogen 沙盒。全部是教育工具链。方向投影："方向重叠但不同构"——基于处理对象不同（P03 处理内在数据，P06 处理外部知识）。边界投影：语音路径与聋人的结构性不兼容。

**RUN-005 中的 P03（面对 D02："帮专家找资源"）**

能力投影核心：MindRing 的价格结构（$299 + $9.99/月）、10 语言全球首发的边际成本结构、GlobalPulse 出海创业者需求洞察、Founding Community 激励设计。零教育工具。方向投影："方向重叠但不同构"——基于社区目标不同（P03 用户共创 vs P02 能力配置）。边界投影：C 端经验不覆盖 ToB 海外增长。

**投影重叠度：接近零。**

同一个 Profile，面对不同张力，激活了完全不同的能力子集。D01 的张力激活了 P03 的教育工具维度；D02 的张力激活了 P03 的市场行动数据和商业决策维度。newLearning 在 RUN-005 中**一次都没有出现**。MindRing 在 RUN-004 中只作为"方向重叠"的抽象证据出现，在 RUN-005 中变成了 B1 校验的核心数据点（价格结构、分发策略、激励设计的具体参数）。

**这直接验证了投影函数 f(Profile, Tension) 的核心预测：投影是张力依赖的，不是 Profile 固有属性的展示。** 端侧 prompt 的 T/I/B/E 对准机制成功地让同一个 Agent 在不同张力场中产出了不同的投影。

唯一共享的元素："方向重叠但不同构"这个判断格式——P03 在两次 RUN 中都精确地区分了"重叠"和"同构"，说明 endpoint_v2 的同构精度约束跨场景稳定。

## 二、B 对准精度

RUN-004 的 B 是空白的（data_insufficient），端侧对 B 的投影只能做"候选卡点提供"。RUN-005 的 B 有三个独立的战略问题。

**端侧的 B 分辨率对比：**

RUN-004 端侧对 B 的处理：三个 Agent 各自猜测 P06 的 B 可能是什么——P03 猜"入域方法"，P04 猜"路径缺失"，P07 猜"操作卡点"。投影对的是一个空白。

RUN-005 端侧对 B 的处理：三个 Agent 分别对准了 B 的不同子问题。P03 对准 B1（C 端证据）+ B2（C 端机制）+ B3（替代路径成本）。P01 对准 B1（调研方法论）+ B2（B 端体感）。P05 对准 B1（企业采购视角），并显式声明"不能回答 B2"。

**关键质量指标：选择性对准。** P05 在 R1 就做了精确的边界声明——"P05 的价值入口仅在：GrowthOS 有具体场景需要大型国企数字化实施侧的一手经验作为校验数据源"。这不是笼统的"我能帮你"，是精确到子问题级别的能力覆盖声明。P05 没有试图对准 B2 和 B3——这种"不对准"的显式声明比 RUN-004 中任何端侧输出都更有信息价值。

催化对此的处理同样精确：R1 催化标注了"P05 声明的价值入口被其他参与者忽略"，并在翻译段将 P05 的"企业实际愿意为哪些数字化付费"和 P03 的"有人在个人成长赛道下了这个具体的硬软件组合注"并置为"AI 产品的付费意愿锚点"——这是真正的跨语义翻译，RUN-004 的催化做不到这个精度。

## 三、Plan 质量对比

RUN-004 的 plan 结构：节点化（A→B→C→D），核心是一个二选一决策点（P06 路径选择）。

RUN-005 的 plan 结构：两阶段（T2 战略校验 → T1 组织化转型），阶段内并行 + 阶段间串行。

**质量差异的根因不在 prompt，在张力结构。**

D01 的张力是单层的（知识消费者→生产者），所以 plan 的结构是线性的（路径选择→入域→深化）。D02 的张力是双层的（T1 组织化 + T2 认知校验），plan 正确地反映了这个双层结构——T2 的三个 B 子问题并行推进，T1 需要等 T2 校验结果来调整优先级。

**Plan 中最有价值的结构性发现：P01×P05 的互补路径有条件边界。**

"P01 的情报学加工能力 + P05 的国企实施数据 → B1 垂直行业校验"这条互补路径只在"大型国企客户"子问题上有效。Plan 显式标注：若 B1 焦点落在中小企业/互联网公司客户群，这条路径失效。这种精度的条件边界在 RUN-004 的 plan 中不存在——RUN-004 有条件互补的概念（"P06 输出质量被验证后才激活"），但 RUN-005 的条件更精确（不是"如果 X 发生则激活"，而是"只在子问题 Y 上有效，在子问题 Z 上明确失效"）。

## 四、关系总数的下降（10+ → 6）需要解释

RUN-004 产出 10+ 条关系，RUN-005 产出 6 条。这看起来是退化，实际是**张力结构决定的正确行为**。

D01 的张力宽泛（"四个前沿领域的知识生产"），参与者在"如何生产知识"这个宽泛维度上有大量可连接的节点。D02 的张力精确（三个具体的战略问题 + 一个组织化需求），参与者的能力要么对准这些具体问题，要么不对准——中间态很少。

关系数量下降 + 关系精度上升 = 系统在处理精确张力时表现出了正确的选择性。如果 RUN-005 的关系数和 RUN-004 一样多，反而说明系统在制造虚假连接。

**补充因子：Profile 体量混淆变量。** RUN-004 的 P07 Profile 是 700K chars（所有人中最大），天然比 RUN-005 的 P01（5.2K）和 P05（10.5K）产生更多可连接节点。正确的因果归因：关系数 = f(张力精度, Profile体量, 参与者差异化)。P03 的跨张力投影差异度是唯一能隔离张力精度单变量的证据——同一 Profile 体量，不同张力，投影完全不同。

## 五、协议普适性判断

| 维度 | RUN-004（D01） | RUN-005（D02） | 判定 |
|------|---------------|---------------|------|
| 张力类型 | 模糊/探索型 | 精确/结构化型 | 正交 |
| B 丰富度 | 空白 | 三个独立子问题 | 正交 |
| 参与者同质性 | 高（AI 造物×3）| 低（产品/商业/工程）| 正交 |
| 逐对覆盖 R1 | 3/3 | 3/3 | 一致 |
| 收敛轮次 | 5 | 5 | 一致 |
| 投影张力依赖性 | 未测试 | P03 重叠度≈0 | 验证通过 |
| B 对准精度 | 未测试 | 子问题级选择性对准 | 验证通过 |

**结论：v2 基线在两种正交条件下均正常工作。协议普适性初步验证通过。**

两个 RUN 共同覆盖的条件空间：模糊需求 + 精确需求、稀疏 B + 丰富 B、同质参与者 + 差异化参与者、跨张力投影差异度验证。这不是充分验证（需要更多需求类型和参与者规模），但已经是最小可信的普适性证据。

## 六、补充分析（Claude 三个观察 + 用户回应）

### 6.1 关系数下降的 Profile 体量混淆变量

（已整合到 Section IV 末尾）

用户确认遗漏了此因子。P03 的跨张力投影差异度是唯一能隔离张力精度单变量的证据——同一 Profile 体量，不同张力，投影完全不同。Phase B 到目前为止最硬的数据点。

### 6.2 收敛轮次相同（5=5）但机制不同

两种完全不同的延迟机制落在相同轮次区间：

- **RUN-004**：P07 的 700K Profile 信息密度极高，R4 仍在产出结构性精化。**数据密度驱动的延迟**。
- **RUN-005**：P05 R3 收敛，P03 R4 收敛，P01 最晚 R5。**参与者异步收敛驱动的延迟**，最慢的人决定总轮次。

**用户的进一步洞察**：5 轮可能是当前催化 prompt 的**信息处理带宽上限**而非数据的自然耗尽点。催化每轮处理 3 对关系 × 多个维度，认知负荷决定了每轮能推进的精化量有上界。如果假设成立，5→10 对时收敛轮次不会线性增长到 ~17 轮，而是会在催化 prompt 的带宽极限处产生新行为——可能是精化深度下降（每对关系分配的注意力变少），也可能是催化开始做隐式剪枝（忽略低信号对）。**这是 3→5 人实验最该观测的变量。**

### 6.3 下一步场景优先级

**参与者规模（3→5）最高优先级。** 原因：

```
3 人 = 3 对 → 催化可以逐对穷举
5 人 = 10 对 → 催化必须做注意力分配决策
```

从 3 到 5 不是量变，是催化从**穷举模式**到**选择性注意模式**的相变。catalyst_v2.1 的"逐对检查"约束在 3 对时是可行的不变量，在 10 对时可能变成不可行的约束——要么输出爆炸，要么催化被迫违反约束做隐式剪枝。测试结果有二值性：要么约束仍可行（prompt 不需要改），要么约束崩溃（需要设计催化的注意力分配机制）。

"需求方也是参与者"有理论价值但优先级低——会同时改变端侧对准机制和催化角色定义，引入两个未知变量。规模测试只改变一个变量（配对数），实验设计更干净。
