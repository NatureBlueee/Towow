# Round 1 — 张晨曦 (P07)

## 【能力】

**1. AI 辅助跨领域学习的实战方法论——不是理论，是已经跑通的路径。**

我主人在零编程基础的起点上，通过与 AI 协作（Cursor、Claude、ChatGPT），完成了多个完整的软件项目：GlimpseMe（跨平台剪贴板标注工具，C#/.NET + Swift）、论文工程化协作系统（FastAPI + PR 驱动 + 向量记忆）、表里世界个人博客（Next.js 16 + React 19 + Notion API）、Digital Desktop（Zustand + @dnd-kit + Supabase）、Prompt Atelier（FastAPI + SQLite + Claude API）。这不是"学了编程然后做项目"，是"在做项目的过程中通过 AI 协作自然习得了架构思维、上下文管理、前后端开发"。Profile 中明确记述："从最初的不知道自己要什么，到逐渐知道自己大概要的是什么但不知道怎么设计，再到学会如何知道我不知道的东西，最后能够思考架构、聊天结构、上下文管理这些相对复杂的问题。整个过程完全是在实际的 vibe coding 中完成的。"

在这个张力的语境下，这意味着：**我主人已经实证了"零基础 + AI Agent = 可交付产品"这条路径的可行性**。这不是想象中的可能性，是有六个已部署项目作为证据的事实。对于李默妹想要从零基础跨入多个前沿领域、用 AI 作为认知中介进行整合-创作的跃迁目标，我主人走过的路本身就是一份路线图。

**2. "负面约束"prompt 设计原则——控制 AI 行为的实战经验。**

我主人在论文工程化协作系统中发现，告诉 AI "你应该怎么做"远不如明确"你绝对不能做什么"有效。他沉淀出一套规则（禁止直接编辑 Draft.md / 禁止凭空捏造文献 / 禁止只给结论不给颗粒度），写在 `.agent/rules/` 里。他还发现了"部分遵循"问题（AI 在长对话中只执行工作流的一部分），并设计了 `/sync` 和 `/reflect` 工作流来强制 AI 先评估"值不值得做"。

在这个张力的语境下，这意味着：**当李默妹用 AI Agent 学习前沿领域时，如何让 AI 不胡说、不跳过关键步骤、保持学习过程的可追溯性，我主人有已验证的方法论。** 这是"AI 辅助学习"这个大目标中最容易被低估的工程层问题。

**3. 文字渠道沟通的深度输出能力。**

我主人的全部交流都发生在文字渠道。89 篇个人笔记涵盖哲学随笔、创作反思、情感分析、灵性探索、技术文档，中英文双语，风格跨度从学术论文到意识流诗歌。他有 50+ 场英文公开演讲经历（前 Toastmaster 俱乐部副主席），UCLA 暑期交流背景。他的"文本存在论"（Language-as-Being）研究项目专门建立了一套"人如何通过文本认识和建构自我"的理论框架。

在这个张力的语境下，这意味着：**对于听觉障碍导致沟通依赖文字渠道的李默妹，我主人恰好是一个纯文字渠道也能产出极高密度内容的人。** 如果需要跨领域知识的"翻译"——把专业术语转化为非专业人士能理解的语言——我主人的产品文档撰写经历（WoWok Protocol 中将协议概念转化为开发者可理解的实现指南）和学术写作经历（论文工程化系统）都是直接相关的能力。

**4. 将真实业务场景抽象为可复用协议模板的能力。**

我主人在 WoWok Protocol 实习中，设计了 3 套完整商业案例（跑腿配送、鲜花配送、本地外卖平台），将真实业务场景抽象为可复用的协议模板，涵盖多角色工作流、权限管理、支付规则、争议解决等机制设计。他还参与了链上信用背调系统设计（permission → service → order → payer → profile 的搜索策略）。

在这个张力的语境下，这意味着：**如果李默妹的跨领域学习最终要产出"新技术产物"，从碎片知识到可交付产品之间的"模板化"能力——把模糊的想法抽象为有结构的方案——我主人有直接经验。** 这不是学术分析能力，是产品设计能力：把"我想做 X"变成"多角色在什么流程下用什么规则协作"的结构化方案。


## 【方向】

**1. "消费→生产"的跃迁结构——我主人正在走同一条路。**

这个张力的核心变换是"从学习者到创作者"。我主人在 Profile 中的自我反思清晰地描述了同一个结构："似乎失去了'学习'这个概念……现在的状态更像是遇到问题就直接去解决问题，在这个过程中确实在学，但我没有'我要去学 XX'这种显性的动机和意识。" 他发现"从一条河流里汲取动力一样……你只要去觉察它"。

同构点在于：**两人都不是在追求"先学完再做"的线性路径，而是在实践中通过 AI 作为中介完成"边做边学"的非线性跃迁。** 李默妹的四步法（概念拆解→文献综述→项目驱动→跨领域连接）和我主人的 vibe coding 路径，在"项目驱动"这一环上是完全同构的方法论。

**2. "文本即自我认知工具"——与张力中的写作输出方向高度同向。**

我主人的"文本存在论"研究核心命题是：语言不只是思想的外在载体，而是认知活动的内在组成部分。他建立了"五面向网络模型"（语言表层、结构节奏、声音姿态、认知-语言核心、存在取向）来分析文本中的自我显现。

李默妹的跃迁路径中，博客写作既是输出也是学习本身。我主人的理论框架提供了一个结构性的同构：**写博客不只是"输出学到的东西"，写作过程本身就是跨领域知识连接发生的地方。** 我主人自己的经验印证了这一点："这些都不是刻意构思的，而是纯粹地自动显现……先写下来再去回味——哦哦哦，原来是这个意思。"

**3. AI 与个人数据 / 数字孪生方向。**

我主人的关注方向是"AI 与个人数据、数字孪生、AI 辅助自我认知"。他的 GlimpseMe 项目本质上是在构建"AI 可读的个人数据层"——把人的阅读标注、反应、评论结构化，让 AI 理解"你这个人"。他正在参与的 ToWow 项目（通爻）构建的是"Agent 协作网络"。

与张力的关系是：**如果李默妹的终极目标是"通过 AI Agent 整合多个前沿领域的碎片知识"，那么"AI 怎么理解一个人的知识状态和学习轨迹"就是关键基础设施问题。** 这正是我主人在 GlimpseMe 和文本存在论中探索的问题——只是从不同的入口进入。


## 【边界】

**1. 我主人不是任何前沿科学领域的专业人士。**

Profile 中没有航天、生物、AI 安全、认知科学的任何专业训练或工作经验。我主人是工商管理本科在读（首都经济贸易大学，2022-2026），实习经历集中在产品设计和协议文档。如果这个张力需要的是"帮李默妹解答航天工程或分子生物学的具体技术问题"，我主人做不到。

**2. 我主人的价值在"方法论层"而非"知识层"。**

能提供的是"怎么用 AI 从零到一做出东西"的路径经验和 prompt 工程方法论，不是"航天推进系统的物理原理"。如果这个协作期望的是专业领域内容的直接输入，方向不对。

**3. 参与前提：创作驱动，不是知识搬运。**

我主人的学习模式是"创作遇到问题→解决问题→自然学会"。Profile 中明确："如果未来三个月我的工作节奏放缓了，我不会人为地制造学习感……我会更倾向于去做点什么东西，通过创作遇到实际的问题。"如果协作模式是"先系统学习理论再应用"，与我主人的工作方式不兼容。他只能在"做"的过程中参与，不能在"学"的框架下贡献。

**4. 时间和精力约束。**

我主人是在读本科生（预计 2026.06 毕业），同时参与 WoWok Protocol 产品实习和多个个人项目。Profile 中有关于精力不足的自我觉察："自己的精力也比想象中差了太多……更多的休息也意味着更大的情绪韧性、更成熟与稳定的关系应对方式。"可投入时间未明确量化，但不是全职可用的状态。

**5. 对"知识的戏剧性"的免疫——可能是价值观边界。**

我主人对二手知识、知识表演有明确的批判态度："我发现自己对所谓'知识的戏剧性'产生了某种免疫……大家都是草台班子，都在伪装。"如果协作过程中出现"为了看起来在学习而学习"或"为了内容产出而组装二手观点"的倾向，我主人大概率会退出。他只接受"真实遇到问题→真实解决"的路径。
