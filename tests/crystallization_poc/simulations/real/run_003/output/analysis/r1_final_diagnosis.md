## 核心诊断

### 病因

endpoint_v1 得了 formulation_v0 的同一种病：**膨胀**。

投影函数的本质是压缩：Profile × 张力 → 最小必要的关系信号。但没有密度约束时，LLM 默认膨胀：Profile UNION 张力注释 → 尽量多说。

### 因果链

```
膨胀（能力投影 70-80% 是 Profile 事实列举）
  → 同质性（三人都膨胀了同一维度——零基础造物）
    → 催化翻译退化（R1 只能做术语归类，不能跨语义）
  → 独特能力被淹没（P04 的组织动员排末位、语境化最弱）
    → 催化注意力偏斜（P04↔P03 零直接双边关系）

命名具象化（输出中叫"李默妹"）
  → 服务框架激活（"我能帮你做什么"而非"我有什么+意味着什么"）
    → 加剧膨胀
```

### 正确的处方

**不是"压缩"（描述输出形态），是"只说改变下游操作的东西"（定义输出函数）。**

信息瓶颈原理的操作化：端侧输出中每句话的价值 = 它对催化关系识别的边际贡献。

- "我主人有 10+ 独立项目，覆盖 Swift / TS / React / Vue..." → 催化看了，关系识别不变。**无价值。**
- "我主人在三个产品中反复构建情绪感知→干预→恢复闭环，这在跨领域学习的高挫败过程中变成了心理稳定性的支撑工具" → 催化看了，立即识别出宝盖头↔张力 B 的互补。**有价值。**

区别不在长短。在于**是否改变下游操作**。

| 优先级 | 改什么 | 怎么改 | 解决的问题 |

### 不修的东西及原因

- **推断性边界**（subagent 观察 3）：不修。P07 的"知识戏剧性免疫→大概率退出"推断产出了全轮最高价值的对冲识别。催化自己能区分硬事实和推断。**不要约束正在产生价值的行为。**
- **R1 催化翻译质量偏低**：不修。信息动力学的自然特征（R1 广度轮 → R2 深度轮自然恢复）。
- **P04 方向投影个别连接推理偏弱**：不修。数据层问题（Profile 结构清晰度），prompt 改不了。

## 方法论沉淀

### 架构级 vs 优化级的分层判断

- **架构级问题**（范式错误）→ 用不变量审计：从症状找到被违反的不变量，修不变量
  - 例：formulation v0 的"膨胀为心理画像"→ 范式从"丰富化"改为"编码"
  - 例：endpoint v0 的"参加讨论激活 cosplay"→ 范式从"对话"改为"投影"

- **优化级问题**（范式对但有系统性损耗）→ 用信息函数审计：只修系统性损耗信息函数的行为
  - 例：endpoint v1 的"膨胀"→ 不是范式错误（三重投影是对的），是信息密度约束缺失
  - 处方：不加 7 条分散约束，加 1 条信息函数约束（"这句话改变下游操作吗？"）

### 修 prompt 前的必要检查

每条新增约束，问：**这条约束会不会杀死某个有价值的涌现？**

- 如果会 → 不加这条约束，或换成更窄的表述
- 如果不会 → 加
- 如果不确定 → 在下一个 run 中观察，不提前加

### 一条不变量覆盖一类问题

好的不变量是一个**函数级声明**（"输出只包含改变下游操作的信息"），不是**症状级禁令**（"不要列简历""不要松散类比""不要均匀列举"）。前者让 LLM 自己做信息筛选，后者把筛选责任留给了禁令列表——列表永远追不上退化模式。
