# 协作方案 — D01

## 一句话

三个方法论者为一个跨领域自学者搭建"认知过程可见性系统"——让"不知道自己不知道什么"变成可检测、可干预、可交付的信号。

## 分工

### 李默妹（需求方 / P06）
- 角色: 唯一用户 + 数据源。她的学习行为本身就是这个协作的输入数据和验证场景。
- 交付物: 每个微周期产出微凭证（结构化笔记、PR 式修改建议、Reference 整理、Consensus 讨论记录），而非传统博客文章。微凭证是学习行为的可验证副产品。
- 获得: 一套完整的"从零基础到可交付产出"的方法论支持体系——包括 AI 协作工程（正面构造 + 负面约束）、元认知诊断（"你在哪卡住了"+"你为什么告诉自己没卡住"）、以及把写作过程变成认知工具的具体方法。
- 投入: 持续的学习行为 + 愿意让学习过程被工具捕获和诊断（写作断裂点、修改模式、空白期等过程数据将被用于反馈闭环）。

### 宝盖头（P03）
- 角色: 数据捕获工具设计者 + 产品化判断者。负责元认知诊断的"数据采集层"——把不可见的认知过程变成可检测的信号。
- 交付物:
  1. 写作过程数据检测机制的设计方案（写作断裂点检测、反复修改模式识别、空白期标记——从 MindRing 反馈闭环设计原则解耦出的文本版本）
  2. AI Prompt 正面构造资产（知识树生成器、概念讲解 prompt、autogen 沙盒配置）供李默妹直接使用
  3. 产品周期经验判断：什么时候一个微周期的产出够格成为"产品级"交付物
- 获得: 一个真实的、非自己的用户场景来验证"反馈闭环方法论"在产品设计以外领域的迁移性；聋人用户的使用数据填补 MindRing 在无障碍场景的设计盲区。
- 投入: 设计方案的时间（不是写代码，是输出设计原则和具体 prompt）；对自身设计直觉盲区的持续校正（Round 3 已展示这种能力——主动承认语音输入不兼容并提出替代路径）。

### Mark Jin（P04）
- 角色: 微周期流程协调者 + 行为层数据采集者。不设定时间约束，而是设定反馈节奏——每个微周期（2-4 周）的开启条件是"上一个问题成熟了"，不是"日历到了"。
- 交付物:
  1. 微周期协调流程：每个周期的启动信号（问题成熟度判断）、检查点（微凭证产出）、反馈回收（过程数据诊断结果）
  2. 微凭证与微周期的对接规则：每个微周期的交付物就是该周期的微凭证，Mark Jin 负责确认"这个微凭证是否足够具体可验证"
  3. 生理信号采集层的可行性评估（脑电经验的间接贡献——评估是否以及如何将生理数据纳入认知诊断，不是必须做，是评估做不做）
- 获得: 一个实际的微周期协调实践案例（从竞赛组织到学习支持的场景迁移）；与宝盖头和张晨曦的方法论工具链整合经验。
- 投入: 每个微周期的启动/检查/回收三个节点的协调时间；对"可衡量交付物"标准的灵活性（接受微凭证作为交付物，而非传统的项目产品）。

### 张晨曦（P07）
- 角色: 认知层诊断者 + 协作结构设计者。负责元认知诊断的"解释防御层"——识别学习者的合理化模式和认知防御；同时设计三人协作本身的运转结构。
- 交付物:
  1. 微凭证格式规范（什么样的结构化笔记算"可验证"，什么样的 Reference 整理算"有质量底线"——质量通过格式约束实现，不通过频率限制实现）
  2. AI 协作负面约束清单（禁止直接编辑、禁止捏造文献、/reflect 工作流等具体规则）供李默妹配置到自己的 AI 工具中
  3. 思维模式工具（Prompt Atelier 方向）：为每个目标领域预装一个思维框架，作为领域专家缺口的间接补偿
  4. 协作协议模板：三人在同一框架下运转的角色、边界、争议解决规则
- 获得: "写作即认知工具"这个理论命题的一个真实实践案例（从"文本存在论"到具体验证）；WoWok Protocol 设计经验在新场景中的实际应用。
- 投入: 方法论输出的时间（格式规范、约束清单、思维模式工具、协议模板）；对李默妹微凭证的质量审视（不是审批，是"这个产出是否触发了你真正的认知过程"的诊断）。

## 协作节奏

**驱动方式：问题成熟度驱动，不是日历驱动。**

一个微周期的生命周期：

1. **启动信号**：李默妹在某个领域遇到一个具体问题（不是"我要学航天"，而是"我读了这篇文章但不理解 X 概念与 Y 概念的关系"）。Mark Jin 判断这个问题是否足够具体可操作——如果是，启动一个微周期。
2. **执行**：李默妹使用宝盖头提供的正面 prompt 工具 + 张晨曦提供的负面约束规则，通过 AI 协作探索这个问题。写作过程数据被宝盖头设计的检测机制捕获。
3. **检查点**：微周期产出一个微凭证。Mark Jin 确认其可验证性（"有具体产物"），张晨曦确认其质量底线（"格式规范，不是复制粘贴"）。
4. **诊断回收**：宝盖头分析写作过程数据（断裂点在哪、反复修改了什么、哪里空白），回答"你在哪个环节受挫了"。张晨曦分析微凭证内容（是否有合理化模式、是否回避了困难部分），回答"你为什么告诉自己没受挫"。诊断结果回馈给李默妹。
5. **下一周期条件**：基于诊断结果，下一个问题自然浮现。Mark Jin 判断成熟度，启动下一个微周期。

**同步方式**：事件驱动而非会议驱动。每个微周期的检查点和诊断回收是同步点。没有固定的"每周例会"。微周期长度由问题复杂度决定，预期 2-4 周，但不硬性规定。

## 第一步

**最小可执行动作：李默妹选一个具体问题，三人各自输出一份"给李默妹的工具包"。**

具体操作：

1. **李默妹**：从四个目标领域中选一个，写出一个具体的、当前正在困惑的问题（不是"我想学认知科学"，而是"我在读 X 时不理解 Y"）。这就是第一个微周期的启动信号。
2. **宝盖头**：基于这个问题，输出一套可直接使用的 AI prompt（知识树生成器 + 概念讲解 prompt），以及"写作过程数据应该怎么记录"的初版规则（哪些信号要留痕：修改次数、断裂位置、空白时长）。
3. **张晨曦**：输出微凭证格式模板（这个问题的探索结果应该长什么样）+ AI 负面约束清单（配置到李默妹的 AI 工具中）。
4. **Mark Jin**：确认这个问题的微周期预期长度、检查点设定（什么时候看微凭证初稿），以及微凭证的可验证性标准（怎么算"完成了"）。

这四份输出构成第一个微周期的全部前置条件。李默妹拿到工具包后开始执行，产出第一个微凭证，触发第一次诊断回收。

## 残余张力

### 缺口 1：四个目标领域的专业知识（航天/生物/AI安全/认知科学）

三人的能力边界清晰：方法论 + 工程能力 + 元认知工具 + 过程可见性系统。领域知识本身不在覆盖范围内。

**建议处理方式**：
- 短期：张晨曦的思维模式工具（Prompt Atelier）作为间接补偿——为每个领域预装思维框架，让 AI Agent 作为认知中介时有更好的约束。
- 中期：评估"AI Agent + 正确的思维模式工具"是否足够支撑李默妹的跃迁。如果第一个微周期的诊断结果显示领域知识是核心卡点（而非方法论或元认知），则触发新一轮场发现，补入有目标领域背景的参与者。
- 判断标准：如果微凭证的质量问题出在"事实错误"或"概念混淆"而非"学习方法"或"认知防御"，说明领域专家缺口是硬阻碍。

### 缺口 2：聋人无障碍学习渠道

情绪追踪工具已适配文本输入（Round 3 宝盖头的解耦方案），但音视频学习资源对聋人不可用的问题未解决。

**建议处理方式**：在第一个微周期的工具包准备阶段评估——李默妹选的具体问题所需的学习资源中，有多少是音视频？如果大量依赖，需在协作方案外引入技术工具补偿（自动字幕、视频转文本 AI）。这不在三人能力范围内，但可以在工具包中预置推荐方案。

### 缺口 3：张力源的关键数据（B 的卡点记录、E 的时间投入量级）

**建议处理方式**：采纳三人共识——"先动起来，用工具捕获过程数据"。第一个微周期的诊断回收就是这些数据的第一次采集。B 的卡点记录会在写作过程数据中自然涌现，E 的时间投入量级会在微周期执行中自然暴露。不需要预先补齐，执行即补齐。
