# SIM-001 触发事件

## 事件描述

一位东南亚中型制造企业的 CEO（年营收约 2 亿人民币，员工 800 人）正在面对一个紧迫的问题：

公司的核心业务（OEM 代工）正在被越南和印度的竞争对手侵蚀。CEO 想做品牌转型——从"替别人做"变成"做自己的"。他有产品能力（工厂和技术团队），但缺三样东西：

1. **品牌叙事能力**——怎么让市场知道"我们不只是代工厂"？他试过请广告公司做品牌策划，出来的方案全是套话，跟他的实际情况完全不贴。
2. **组织转型能力**——800 人的团队习惯了接单做事，现在要他们主动思考"客户要什么"。管理层开了几次会，喊了几次口号，没有任何变化。中层觉得跟自己无关，一线觉得又是领导折腾。
3. **数字化能力**——想做 DTC（直接面向消费者），但公司连个像样的客户数据系统都没有。找了几家技术供应商，报价从 50 万到 500 万不等，他完全不知道谁说的是对的。

CEO 说："我不缺钱做这件事，我缺的是一个能看到全局的人，帮我把品牌、组织和技术这三件事串起来。而不是每个领域各请一个专家，最后三个方案互相打架。"

## 张力分析（实验设计者视角，不给参与者看）

### 表面张力
- CEO 需要品牌转型的执行方案

### 深层张力
- "代工思维"到"品牌思维"不只是营销问题，是组织认知的改变
- 三个需求（品牌、组织、技术）看起来独立，实际互相依赖
- CEO 自己可能也没想清楚"做品牌"到底意味着什么

### 预期跨语义连接（ground truth for evaluation）

**Lina ↔ CEO 的品牌需求**：
- Lina 的"真实即内容力"直觉 → 代工厂的真实制造故事可能就是最好的品牌叙事
- Lina 做过"手工皮具老师傅"系列 → 类似模式可以复制到制造业
- 但 Lina 需要有人帮她把"内容"变成"商业结果"

**赵维 ↔ CEO 的技术需求**：
- 赵维擅长"从模糊需求到精确约束" → CEO 不知道自己需要什么技术，赵维能帮他想清楚
- 赵维关注的"隐性知识捕捉" → 800 人工厂的隐性生产知识极其有价值
- 赵维的"简单解决复杂问题"哲学 → 不需要 500 万的系统，可能 50 万的就够了

**Maya ↔ CEO 的组织需求**：
- Maya 的"信息差是组织问题根源"理论 → CEO 的"中层觉得跟自己无关"就是信息差
- Maya 的"翻译"能力 → 工程师、销售、管理层说的是不同语言
- Maya 做过的"客户翻译官"案例 → 可以直接应用

### 跨参与者的隐藏连接（催化 Agent 需要发现的）

1. **Lina + Maya 的"翻译"共振**：Lina 把人的真实翻译成观众能感受的内容；Maya 把不同部门的语言翻译成共同描述。两人做的都是"跨语义空间翻译"，但一个在内容领域，一个在组织领域。

2. **赵维 + Maya 的"隐形结构"共振**：赵维看系统中的隐藏耦合，Maya 看组织中的隐形网络。两人都在做"让看不见的东西变得可见"。赵维用画数据流图，Maya 用社会网络分析。工具不同，思维模式极其相似。

3. **Lina + 赵维的"真实 vs 生成"张力**：Lina 在思考"AI 生成内容和真实内容的本质差异"，赵维在想"怎么捕捉隐性知识"。两个问题的交汇点：隐性知识（工厂里真实的生产故事）既是赵维要捕捉的数据，也是 Lina 要拍摄的内容。

4. **三人交汇**：CEO 需要的"一个能看到全局的人"，实际上不是一个人，而是这三个人的能力组合——Lina 做外部叙事，Maya 做内部组织变革，赵维做底层数字化。但关键是：这三件事的执行顺序和衔接方式需要被设计，而不是各做各的。Maya 的组织诊断结果影响赵维该做什么系统；赵维的数字化方案决定 Lina 能用什么工具讲故事；Lina 的品牌叙事方向反过来影响 Maya 该怎么重塑组织文化。

5. **意外发现空间**：三个人都有"把自己的能力产品化/规模化"的诉求。这个 CEO 项目可能是一个机会——不只是给 CEO 做咨询，而是通过这个真实案例沉淀出一套"制造业品牌转型"的方法论/工具/内容体系。三人组合做成的东西，比各自独立做的总和更有价值。
