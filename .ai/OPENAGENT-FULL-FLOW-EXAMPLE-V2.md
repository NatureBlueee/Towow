# ToWow 产品全流程示例 V2 - 清迈小小火人节

> **场景**: 邀请全世界一千个有趣的灵魂，在二月底去清迈办小小火人节
> **目的**: 展示 ToWow 如何处理"非标准化"的、充满个性和随机性的协商场景

---

## 一、场景背景

### 发起者
**小宇** - 一个在清迈定居的数字游民，想在二月底办一场"小小火人节"：
- 不是复制火人节，而是一群有趣的人聚在一起
- 每个人带来自己的"礼物"（才艺、故事、创意）
- 没有门票，但需要"贡献"才能参与
- 地点已经找好了：清迈山里的一个农场

### 核心问题
**什么是"有趣的灵魂"？如何筛选？**

这不是找"会写代码的人"或"有场地的人"，而是找：
- 有故事的人
- 有奇怪爱好的人
- 愿意分享的人
- 能让其他人感到"哇"的人

## 三、完整流程

### Step 0: 小宇提交需求

**小宇通过 SecondMe 输入：**
```
我想在二月底（2月28日-3月2日）在清迈办一场"小小火人节"。

不是要复制火人节，而是想邀请一群有趣的灵魂聚在一起。
场地我找好了：清迈山里一个朋友的农场，可以容纳50-80人。

我不想收门票，但希望每个人都能带来自己的"礼物"——
可以是一个技能、一场表演、一个故事、一顿饭、一堂课，whatever。

最重要的是：我想找真正有趣的人，不是"网红"，不是"成功人士"，
而是那种你坐下来聊天会觉得"哇，这个人好有意思"的人。

我希望这群人来自不同背景、不同国家，
但都有一个共同点：他们在过一种"不太一样"的生活。

对了，我自己可以贡献的是：场地协调 + 做一顿正宗的云南菜。
```

**发出事件:** `towow.demand.submitted`

---

### Step 1: Coordinator 理解需求

**Coordinator 调用 LLM 分析这个"非标准"需求：**

```
这不是一个找"服务提供者"的需求，而是找"灵魂共鸣者"的需求。

关键词提取：
- 核心：有趣的灵魂、不太一样的生活
- 形式：小型聚会、礼物交换、无门票
- 约束：清迈、2月底、50-80人、农场
- 发起者贡献：场地协调、云南菜

筛选维度（不是能力标签，而是灵魂画像）：
1. "有故事"度：这个人有没有做过让人觉得"哇"的事？
2. "分享"意愿：愿不愿意贡献自己的礼物？
3. "非主流"程度：生活方式是否"不太一样"？
4. 地理可达性：二月底能到清迈吗？
5. "氛围"契合：和其他人在一起会不会自然融入？
```

**发出事件:** `towow.demand.understood`

---

### Step 2: Coordinator 筛选"有趣的灵魂"

**这不是 SQL 查询，而是 LLM 阅读每个人的灵魂画像后判断：**

```
LLM Prompt:
你现在是 ToWow 网络的筛选者。
有人想办一场"小小火人节"，找有趣的灵魂参加。

需求描述：
{小宇的需求}

以下是网络中所有用户的灵魂画像：
---
{所有 agent 的 soul_profile}
---

请选出最适合参加这场活动的人。
判断标准：
1. 是否有独特的故事或经历？
2. 能贡献什么"礼物"？礼物有趣吗？
3. 二月底能到清迈吗？
4. 这个人和其他被选中的人在一起，会不会有化学反应？

请输出：
1. 选中的人及理由
2. 为什么他们在一起会很有趣
3. 潜在的"组合亮点"（比如：老王的物理和小鹿的声音疗愈碰撞会怎样？）
```

## 四、核心设计点总结

### 4.1 筛选不是匹配"能力"，而是匹配"灵魂"

```
传统筛选：
- 需要"会做网站的人" → 筛选技能标签

ToWow 筛选：
- 需要"有趣的灵魂" → 筛选灵魂画像
- 考虑因素：故事、独特性、分享意愿、氛围契合
```

### 4.2 Offer 不是"报价"，而是"我是谁"的展示

```
每个人的回应反映了：
- 他们的故事（老王的摩托、小鹿的burnout）
- 他们的怪癖（阿花的厕所收藏）
- 他们的关怀（大熊的小朋友、火星的故事收集）
- 他们的边界（老陈需要安静、小鹿需要静默时间）
```

### 4.3 聚合不是"分配任务"，而是"编排体验"

```
ChannelAdmin 的工作：
- 不是给每个人分配一个时间段
- 而是设计化学反应：
  - 物理+玄学 会碰撞出什么？
  - 慢（茶）+快（音乐）会创造什么氛围？
  - 怎么让陌生人变成朋友？
```

### 4.4 协商是"共创"，不是"讨价还价"

```
Agent 之间的协商：
- 老陈建议虫鸣音乐会换时间 → 不是因为"我不想配合"
- 而是因为"这样整体体验会更好"

所有人都在为共同的目标出谋划策。
```

## 五、给 OpenAgent 开发者的提示

### 5.1 用户画像的维度

```
不只是：
- 技能：["Python", "设计"]
- 位置："北京"

而是：
- 故事：一段经历的描述
- 怪癖：独特的爱好
- 分享：愿意贡献什么
- 氛围：说话风格、给人的感觉
- 边界：需要什么条件
```

### 5.2 LLM 筛选的 Prompt 设计

```
不是问：
"谁有这个能力？"

而是问：
"谁有有趣的故事？"
"谁愿意分享自己？"
"谁和谁在一起会有化学反应？"
```

### 5.3 方案聚合的思维

```
不是甘特图式的任务分配：
- 9:00 A做X
- 10:00 B做Y

而是体验设计：
- 早上需要什么氛围？谁的礼物适合？
- 怎么让陌生人破冰？
- 高潮在哪里？怎么收尾？
```
