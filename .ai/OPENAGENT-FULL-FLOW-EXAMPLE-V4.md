# ToWow 产品全流程示例 V4 - 穷鬼版科切拉（深度版）

> **核心价值**: 展示真实的筛选逻辑、不响应原因、子网递归、礼物经济的多样性
> **场景**: 上海周边的"穷鬼版科切拉"——没有大牌明星，但每个人都是参与者

---

## 一、需求背景

### 发起者：阿灿

```
阿灿，29岁，在上海一家互联网公司做运营。
去年去了一次科切拉，花了3万多，觉得体验很好但太贵了。
回来后一直在想：能不能在国内做一个"穷鬼版"？

他的想法：
- 不请大牌，但每个人都能贡献点什么
- 不是演出，而是参与
- 预算控制在人均500以内（含吃住）
- 规模：100-150人
- 时间：五一假期（4.30-5.2）
- 地点：上海周边2小时车程内
```

### 阿灿提交的需求

```
我想在五一假期（4月30日-5月2日）在上海周边办一场"穷鬼版科切拉"。

不请大牌明星，但希望每个人都能贡献点什么：
- 会乐器的来表演
- 会做饭的来做饭
- 有帐篷的带帐篷
- 啥都没有的就出钱

规模大概100-150人，预算人均500以内（含吃住）。

我自己可以贡献的：
- 整体协调（我做运营的，组织能力还行）
- 我会打碟，可以做一个电音之夜
- 我有一台投影仪，可以做露天电影

目前还没有：
- 场地
- 住宿方案（帐篷？民宿？）
- 餐饮
- 音响设备
- 其他表演者
- 报名系统
```

## 二、选址分析：为什么选上海周边

### Coordinator 的选址分析

```
LLM 分析上海周边的优劣势：

优势：
1. 人口基数大：上海2500万 + 苏州1300万 + 杭州1200万 + 南京950万
2. 消费能力：长三角人均可支配收入全国第一
3. 交通便利：高铁网络发达，2小时内可达多个城市
4. 场地选择多：莫干山、安吉、崇明、嘉善等
5. 年轻人密度高：互联网公司、外企集中

劣势：
1. 五一期间热门景区人多
2. 场地成本可能较高
3. 天气不确定（江南五月可能下雨）

推荐区域（按优先级）：
1. 安吉/莫干山区域：民宿多、自然环境好、距上海2小时
2. 崇明岛：距市区近、场地大、但略无聊
3. 嘉善/西塘周边：交通便利、但商业化严重
```

## 七、不响应分析（重要！）

### 7.1 为什么32人没响应？

**ChannelAdmin 分析不响应原因：**

```
不响应原因分布（32人）：

1. 忙/没看到消息（40%，13人）
   - 五一前加班忙
   - 消息太多没注意
   - 旅行中没信号

2. 不感兴趣（25%，8人）
   - "露营？我不喜欢户外"
   - "100多人？太吵了"
   - "不认识的人聚一起？尬"

3. 有顾虑（20%，6人）
   - "这是什么活动？靠谱吗？"
   - "免费的？有什么猫腻？"
   - "万一是传销呢？"

4. 社恐/内向（10%，3人）
   - 小陈："这种活动不适合我……"
   - "我不太会social……"

5. 已有安排（5%，2人）
   - 五一已经订了别的行程
```


### 15.4 子网递归的必要性

```
没有子网会怎样？

场景1：帐篷不够
- 传统做法：主办方自己想办法买/租
- ToWow 做法：触发子网，找到更多帐篷提供者

场景2：协商中发现新需求
- 老马的帐篷要清洗修补
- 如果没有子网机制，这个需求无法被自动匹配

场景3：边缘角色的参与
- 阿飞（婚庆打工人）不在主网络
- 但他能提供灯光，通过子网被激活

子网 = 让需求找到供给的递归机制
```

## 十六、给 OpenAgent 开发者的启示

### 16.1 筛选逻辑

```
初筛：大范围过滤（地理、时间、基本条件）
二筛：精准匹配（能贡献什么、角色需求）
子网筛选：针对特定缺口的定向搜索

关键：每一层筛选的条件都不同
```

### 16.2 不响应处理

```
需要分析不响应原因：
- 忙/没看到 → 可以再触达
- 不感兴趣 → 不要再打扰
- 有顾虑 → 提供更多信息
- 社恐 → 调整邀请方式

不是所有不响应都是"拒绝"
```

### 16.3 子网触发条件

```
什么时候触发子网？

1. 硬性缺口：没有这个资源活动办不了
   - 帐篷不够 → 必须触发

2. 协商中发现的需求
   - "帐篷需要清洗" → 触发

3. 成本优化
   - "能不能找到更便宜的方案？" → 可选触发

判断逻辑：
- 缺口重要性 > 70% → 触发
- 主网络无法满足 → 触发
- 预期收益 > 触发成本 → 触发
```
