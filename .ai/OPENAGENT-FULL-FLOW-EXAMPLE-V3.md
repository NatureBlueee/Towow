# ToWow 产品全流程示例 V3 - 清迈小小火人节（涌现版）

> **核心价值**: 不是找"有趣的人来表演"，而是**激发每个人发现自己能贡献什么**
> **涌现**: AI撮合出人们原本不会想到的合作

---

## 一、场景：这不只是一场活动，而是一个复杂的协作网络

### 发起者的需求
**小宇** 想在清迈办小小火人节，但他只有一个想法，没有：
- 场地
- 住宿
- 餐饮
- 交通
- 物料（帐篷、音响、灯光）
- 政府许可
- 报名系统
- 医疗保障
- 翻译（有国际参与者）
- 记录和传播
- ……

**这需要几十个人的协作，每个人贡献不同的东西。**

## 三、完整流程：涌现式协作

### Step 0: 小宇提交需求

```
我想在二月底在清迈办一场"小小火人节"。
规模大概50-80人，持续3天。

我目前只有一个想法，其他什么都没有：
- 没有场地
- 没有住宿方案
- 没有餐饮
- 没有设备
- 不知道需要什么许可
- 没有报名系统
- ……

我希望这是一场共创的活动，每个人都贡献自己能贡献的。
不是我请人来"服务"，而是大家一起"参与"。

我自己可以贡献的：整体协调 + 做一顿云南菜。
```

---

### Step 1-2: Coordinator 分析需求，识别需要的角色

**Coordinator 不是找"有趣的人"，而是分析这个活动需要什么：**

```
活动需求分析：

核心需求：
1. 【场地】50-80人，3天，最好在自然环境中
2. 【住宿】参与者需要住的地方
3. 【餐饮】3天的吃饭问题
4. 【交通】从清迈市区到场地
5. 【设备】音响、灯光、帐篷、发电机
6. 【许可】泰国办活动需要什么手续？
7. 【医疗】安全保障
8. 【报名】需要一个报名和管理系统
9. 【宣传】怎么让合适的人知道这个活动？
10. 【记录】拍照、视频
11. 【内容】活动本身的内容和体验
12. 【翻译】可能有国际参与者

角色需求：
- 场地提供者
- 住宿协调者
- 餐饮提供者
- 交通组织者
- 设备提供者
- 政府关系/许可
- 医疗保障
- 技术支持（报名系统）
- 营销推广
- 摄影记录
- 内容贡献者（表演、workshop等）
- 翻译
```

---

### Step 3: Coordinator 筛选——不是找"最合适的"，而是找"可能有贡献的"

**关键区别：AI 不是替人决定谁来参与，而是识别谁"可能"能贡献什么**

```
筛选逻辑：

1. 老曾（农场主）
   - 表面：他是卖菜的
   - AI发现：他的农场有空地，可能可以作为场地
   - 邀请理由：问问他的农场能不能办活动？

2. 阿美（餐厅老板）
   - 表面：她开小餐厅
   - AI发现：她有团队，可能可以做大型餐饮
   - 邀请理由：问问她能不能做100人的餐

3. Kevin（产品经理）
   - 表面：他是打工人，来清迈度假
   - AI发现：他会用AI工具，可能可以快速搭建报名系统
   - 邀请理由：问问他有没有兴趣做技术支持？

4. 小莲（青旅老板）
   - 表面：她开青旅
   - AI发现：她认识很多住宿资源
   - 邀请理由：问问她能不能协调住宿？

5. 阿坤（司机）
   - 表面：他是开车的
   - AI发现：他认识所有司机，还认识村长
   - 邀请理由：交通组织 + 可能帮忙搞定当地关系

6. 大卫（音响租赁）
   - 表面：他租音响
   - AI发现：他参加过真正的火人节，懂设备
   - 邀请理由：不只是租设备，还可以当顾问

7. 小林（护士）
   - 表面：她是护士
   - AI发现：她可以做医疗保障
   - 邀请理由：安全第一

8. 阿泰（导游）
   - 表面：他是导游
   - AI发现：他懂政府流程
   - 邀请理由：搞定许可

9. 老张（摄影师）
   - 表面：他退休了
   - AI发现：他是专业摄影师，可能愿意记录有意义的事
   - 邀请理由：问问他有没有兴趣？

10. 小雪（营销）
    - 表面：她做营销

    - 邀请理由：宣传和报名推广

    - 表面：他是表演者
    - AI发现：他认识很多音乐人
```

### Step 4: 广播邀请——不是说"你来参加"，而是说"你可能能贡献什么"

**每个人收到的邀请是不同的、针对性的：**

**给老曾（农场主）的邀请：**
```
老曾你好！

有人想在清迈办一场小型聚会活动（50-80人，3天）。
我注意到你在山里有一个农场，想问问：

1. 你的农场有没有可能办这种活动？
2. 如果可以，条件是什么？
3. 你愿意以什么方式参与？（提供场地？参与活动？）

这不是商业租场地，而是一群人共创的活动。
如果你感兴趣，可以告诉我你的想法。
```

**给Kevin（产品经理）的邀请：**
```
Kevin 你好！

有人想在清迈办一场活动，需要一个报名系统。
我看到你是产品经理，而且在清迈remote。

想问问：
1. 你有没有兴趣花一个周末搭一个简单的报名网站？
2. 如果参与，你希望以什么方式？（纯技术支持？还是也想参加活动？）

这不是外包工作，而是一起共创。
参与的人都会贡献自己的一份。
```

**给老张（退休摄影师）的邀请：**
```
张老师您好！

有人想在清迈办一场有意思的活动——小小火人节。
我知道您以前是新闻摄影师，退休了。

想问问：
1. 这种活动您有兴趣来拍吗？
2. 如果来，您希望怎么参与？

不是请您来"工作"，而是邀请您来记录一些可能有意义的东西。
如果您觉得没意思，完全可以拒绝。
```

**关键：AI 帮助每个人"发现"自己能做什么，而不是"表演"自己是什么**

## 四、核心价值总结

### 4.1 不是"找人来服务"，而是"激发贡献"

```
传统模式：
- 我需要场地 → 找场地租赁
- 我需要餐饮 → 找餐饮公司
- 我需要音响 → 找设备供应商

ToWow 模式：
- 老曾从来没想过自己的农场可以办活动
- Kevin 从来没想过可以用技术参与一个线下活动
- 老张从来没想过退休后还能做有意义的摄影
- AI 帮他们发现了自己的贡献可能性
```

### 4.2 涌现式合作

```
没有人计划这些事情，但它们自然发生了：
- 阿美 + 野子 → 音乐晚餐
- 阿坤 + 村长 → 本地人参与
- 老曾老婆 → 潮州菜workshop
- 小林 → 急救培训变成内容
- Kevin → 弹幕互动
- 大卫 → "leave no trace"的精神
```

### 4.3 每个人都不只是"那个角色"

```
老曾：不只是场地提供者，老婆变成了workshop老师
阿坤：不只是司机，成了当地连接者
大卫：不只是设备供应商，成了火人节精神顾问
老张：不只是摄影师，成了故事记录者
小林：不只是医疗保障，成了急救培训师
Kevin：不只是技术支持，创造了弹幕互动体验
```

## 五、给 OpenAgent 开发者的提示

### 5.1 邀请不是"你来做这个"，而是"你可能能贡献什么"

```
传统：
"我们需要一个餐饮供应商，你来报价。"

ToWow：
"有人想办活动，我注意到你有餐饮经验。你有没有想法？"
→ 让对方自己发现能贡献什么
```

### 5.2 聚合不是排日程，而是发现化学反应

```
LLM 的工作：
- 分析所有人的回应
- 发现谁和谁可能碰撞出新东西
- 主动建议合作可能性
```

### 5.3 协商是共创，不是讨价还价

```
协商中会发生：
- A 说"我想和 B 聊聊"
- B 说"我认识 C，可以拉进来"
- 新的想法不断涌现
→ 最终方案是所有人共创的，不是组织者单方面设计的
```

---

**这才是 ToWow 的价值：**
**不是撮合"供应商"，而是激发每个人发现自己能贡献什么，然后产生意想不到的合作。**
